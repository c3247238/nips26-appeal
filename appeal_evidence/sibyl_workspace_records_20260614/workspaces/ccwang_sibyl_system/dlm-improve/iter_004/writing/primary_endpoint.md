# Primary Endpoint Note

## Primary Endpoint

The primary endpoint of the current paper is:

> A bounded full-scale comparison on GSM8K asking whether `cand_espd` yields a better quality-speed trade-off than the matched fixed-frontier sham under a unified runtime contract.

## Secondary Endpoints

- Comparison against shared controls (`CARD-84`, `RAND-84`)
- Paired repair/harm decomposition
- Runtime-lineage transparency
- Stopping-behavior analysis

## Non-Endpoints

- Benchmark-level SOTA
- Cross-task generalization
- Validation of object-level repair hypotheses
- Strong significance-based dominance claims
