# Result Debate Synthesis

**Synthesizer**: main control plane  
**Date**: 2026-03-12  
**Inputs**: `optimist`, `skeptic`, `strategist`, `methodologist`, `comparativist`, `revisionist`

## 1. 最终共识

六个视角已经收敛到同一条主线：当前 iteration 3 最该继续推进的，不是重新寻找正向 controller 故事，而是把已经闭环的 `cand_negative_audit_pivot` 写成一篇 **audited negative case study**。

共识点主要有四条：

1. `CARD-84` 相对 `DNB-84` 确实存在局部 GSM8K 修复信号，但这不足以单独支撑正向方法主张。
2. `RAND-84` 这条 sham control 会实质改变解释，因此 negative result 不是附属材料，而是论文核心。
3. 当前最有价值的 supporting contribution 是最小审计资产包：
   - matched-compute baseline
   - sham control
   - sample-level audit
   - current-only artifact closure
4. 后续最优动作不是加实验追方法胜利，而是把写作边界、图表顺序和 reviewer 防守资产彻底收紧。

## 2. 六方分歧如何被解决

### 2.1 optimist vs skeptic

- `optimist` 强调：局部信号、artifact closure 与 minimal audit template 已足以构成一篇有价值的负案例论文。
- `skeptic` 强调：这仍然只是 audited slice，任何把 `CARD-84` 写回 winning method、把 entropy 写成 targeting rule 的话术都会失真。

综合后，最终版本保留乐观方对“价值”的判断，但完全接受怀疑方对“措辞上限”的约束：

- 允许写 localized signal
- 不允许写 validated efficacy

### 2.2 strategist vs revisionist

- `strategist` 要求明确 `PROCEED`，但这个 `PROCEED` 只能指向当前 negative-case lane，不是回到旧正线。
- `revisionist` 则进一步指出：本轮真正成功的不是实验本身，而是 paper object 已经被正确 pivot。

综合后，我们把后续推进定义为：

- `PROCEED on the negative-case lane`
- `Do not reopen the old method-forward framing`

### 2.3 methodologist vs comparativist

- `methodologist` 强调方法学价值来自 stronger control design 与 asset closure，而不是单个分数。
- `comparativist` 强调对外定位必须是 interpretation-correction / audit contribution，而不是 another method paper。

综合后，论文外部定位被压成：

- compute-controlled, sham-audited negative case
- minimal auditable template as supporting contribution

而不是：

- new protocol paradigm
- benchmark standard-setter

## 3. 最终裁决

### 主线

继续推进 `cand_negative_audit_pivot`。

### 论文对象

一篇关于 training-free DLM revision small-gain claim 的 **audited negative case study**。

### 允许保留的最强主张

1. `CARD-84` 对 `DNB-84` 在 GSM8K 上显示出 localized signal。
2. 该 signal 没有与 budget-matched `RAND-84` cleanly separate。
3. entropy 可以被写成 `risk marker`，不能被写成 validated targeting rule。
4. stronger sham control 会改写对 small-gain result 的解释。

### 必须持续避免的主张

1. `CARD-84` is the winning inference method
2. entropy-guided revision reliably improves DLM reasoning
3. observer-guided controller is validated
4. this work establishes a new protocol paradigm
5. the method almost works / is a near-win

## 4. 后续动作优先级

### P0

直接进入后续写作与评审防守整合，不再新增 full-cycle 实验。

### P1

把以下资产变成写作时的硬约束：

1. `claim_scope_map.json` -> wording checklist
2. `figure_specs.md` -> figure order
3. `validation_report.json` -> audited-slice / current-only disclosure
4. `followup_gaps.md` -> future-work boundary

### P2

如果 reviewer 强追问，只允许在 appendix / future work 中讨论：

1. 更强 sham control
2. external prior reinterpretation
3. harm profile refinement

这些 follow-up 不能被写成“当前论文缺失的默认补件”，更不能被写成“CARD-84 差一点就成立”。

## 5. 一句话综合结论

这轮结果最有价值的地方，不是找到了更强的 training-free revision controller，而是用 matched-compute baseline、budget-matched sham control、sample-level audit 和 current-only artifact closure，把一个本来很容易被过度解释的局部增益，改写成了一个可复核、可防守的 audited negative case。
