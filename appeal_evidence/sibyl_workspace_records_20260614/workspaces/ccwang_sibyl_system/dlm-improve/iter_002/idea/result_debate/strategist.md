# 战略家观点

从时间成本与论文收敛角度看，本轮最优策略已经不是再开新实验，而是把已完成资产转成可投稿叙事。

- 继续跑新的 controller/full batch，收益低且会重新引入 story drift；
- 当前最值钱的动作是把 bucket、protocol、seed 三条线组织成统一写作骨架；
- `cand_bucket` 适合做唯一前进候选，`cand_protocol` 作为支撑协议层跟随进入写作；
- `Observer-Controller Split` 只保留在 intro / discussion 的 framing 位置，避免再变成独立贡献。

因此，最合理的决策是：在控制面上继续推进，但实验层面停止扩张，把资源集中到 result synthesis 和 writing。
