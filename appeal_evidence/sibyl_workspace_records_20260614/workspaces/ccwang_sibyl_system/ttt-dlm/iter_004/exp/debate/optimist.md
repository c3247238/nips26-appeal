# 乐观分析者报告：DLM 推理时改进实验结果

**角色**: 乐观分析者
**日期**: 2026-03-08

---

## 一、结果中值得深挖的正面信号

### 1.1 TCR 的 mean PPL 改善值得关注

虽然所有方法在 median PPL 上都未达到统计显著性，但 TCR 呈现出一个被忽视的亮点：**mean PPL 从 30.1 降至 23.8（-20.9%）**。这意味着 TCR 在压制 PPL 异常值（尾部分布的高 PPL prompt）方面可能非常有效。

回顾 Iteration 018 的 pilot 数据也印证了这一点：TCR 在 32 prompt pilot 中 mean PPL 从 30.3 降至 17.9，而 median 变化不大。这说明 TCR 的作用机制不是"普遍提升"，而是**选择性修复不稳定生成**——这恰恰是实际应用中最有价值的能力（用户最关心的是"不要生成特别差的结果"，而不是"平均好一点点"）。

**建议**: 对 101 prompt 进行子集分析，按 vanilla PPL 分位数分组（如 top-25%/mid-50%/bottom-25%），检验 TCR 是否在高 PPL prompt 子集上达到统计显著性。如果确认"TCR 选择性修复差生成"这一叙事，这本身就是一个有趣的发现。

### 1.2 改善比例接近 55% 而非 50%

熵引导重遮蔽的改善 prompt 数为 56/101（55.4%），TCR 为 52/101（51.5%）。虽然不显著，但这不是完全随机的 coin flip：

- 如果方法完全无效，改善比例应精确趋向 50%
- 熵引导的 55.4% 暗示存在**微弱但一致的正方向信号**
- 在更大规模样本（如 500+ prompts）下，这种微弱信号可能变得显著

### 1.3 Diversity 保持良好——排除了退化干扰

所有方法的 diversity >= 0.967，零退化。这与 0.6B MDLM 上 ReMask-Retry 的惨痛教训形成鲜明对比（0.6B 上 adaptive 退化率高达 71%）。这证明 **Dream-7B 的模型容量足以支撑推理时修正而不退化**。退化问题是 scale-dependent 的，在 7B 模型上已不构成障碍。这为后续更复杂的推理时方法（如 IGIR）在 7B 模型上的成功奠定了重要前提。

### 1.4 温度退火的多样性优势

温度退火虽然 PPL 恶化了 8.1%，但多样性最高（0.971 vs baseline 0.967）。在某些应用场景中（如创意写作、头脑风暴），**以少量 PPL 换取更高多样性**是合理的权衡。退火方法零额外计算开销的特性使其在这类场景中仍然有实用价值。

### 1.5 Pilot 中特定 prompt 类型的显著改善

Pilot 的 16 个 prompt 均为简单科学问题（如"解释机器学习"、"描述黑洞的形成"）。在这些 prompt 上，熵引导达到了 -24.9% 的改善。虽然这没有在 full-scale 上复现，但一个值得追问的假设是：**是否存在某类 prompt（如事实性知识问答）对推理时修正特别敏感？**

审视 pilot 的具体 PPL 分布：entropy_r20_mean 在 16 个 prompt 上的 PPL 范围是 7.4~32.9，而 vanilla baseline 是 5.2~63.2。也就是说，熵引导大幅压缩了 PPL 的方差范围。这与 TCR 压制尾部异常值的发现一致——**推理时修正的主要价值可能在于"减少最差情况"而非"提升平均水平"**。

## 二、IGIR 提案的前景评估

### 2.1 IGIR 针对的是正确的失败模式

当前测试的三种方法（TCR、熵引导、温度退火）本质上都是**单轮、贪心、无反馈**的修正：

- TCR：基于轨迹稳定性一次性重遮蔽，没有质量反馈循环
- 熵引导：基于熵信号一次性重遮蔽，没有接受/拒绝机制
- 温度退火：仅修改采样温度，没有选择性修正

而 IGIR 引入了两个本质不同的机制：

1. **Metropolis-Hastings 接受准则**：允许拒绝退化的修正结果。当前方法的核心问题可能不是"选错了位置"，而是"修正后没有验证是否改善了"。MH 机制从根本上改变了这一点。

2. **多轮迭代精炼**：当前方法都是 K=1（单轮修正）。IGIR 允许 K 轮迭代，每轮都基于前一轮的结果进行更精细的调整。在 K=1 效果微弱的情况下，K=5~10 的累积效应可能产生显著差距。

### 2.2 IB-Score 可能比单一信号更有效

TCR 仅用轨迹稳定性（单一信号），熵引导仅用预测熵（单一信号）。IGIR 的 IB-Score 融合了三个互补信号：不确定性 x 轨迹熵 x 梯度范数。这种多信号融合可能比任何单一信号更精确地定位真正需要修正的位置。

从数据看，Dream-7B 的平均轨迹稳定性高达 0.96——这解释了 TCR 为什么效果有限（模型已经很稳定，TCR 找不到足够的不稳定位置）。IB-Score 的梯度范数维度可能识别出轨迹看起来稳定但实际上对全局质量影响最大的位置。

### 2.3 质量评分函数内置抗退化机制

IGIR 的质量评分 Q(y|x) 包含 n-gram 重复惩罚和多样性约束。这意味着即使模型倾向于用重复 token 填充重遮蔽位置，MH 接受准则也会拒绝这种退化输出。这是 ReMask-Retry 在 0.6B 模型上失败的核心教训的直接修复。

### 2.4 理论框架提供了额外价值

即使 IGIR 的实验效果不显著，其理论贡献（IB 统一框架、最优重遮蔽比例闭式界、次模贪心近似保证）本身就具有论文价值。最坏情况下，IGIR 可以转型为一篇"理论框架 + 为什么 training-free 重遮蔽效果有限的系统分析"论文，这在 EMNLP/ACL 仍然有发表空间。

### 2.5 对 IGIR 的务实预期

坦率地说，基于 18 轮迭代的经验，对 IGIR 在 PPL 指标上产生大幅改善的期望应当审慎。但 IGIR 的实验设计有一个关键不同：**它以 benchmark 准确率（GSM8K、MATH500）而非 PPL 作为主要评估指标**。这是一个根本性的改变——PPL 已被证明是 DLM 质量的不可靠代理，而推理任务的准确率是不可欺骗的客观指标。即使 PPL 改善不显著，IGIR 仍可能在准确率上产生改善（或反之），这本身就是有价值的发现。

## 三、负面结果的发表价值

### 3.1 负面结果本身就是贡献

当前的实验结果构成了一个完整的故事：

> "我们系统性地评估了四类 training-free 推理时改进策略（TCR、熵引导重遮蔽、温度退火、并行投票）在 Dream-7B 上的效果，发现在 101 prompts x 3 seeds 的规模下，没有任何方法产生统计显著的 PPL 改善。这与我们在 0.6B MDLM 上的负面发现（ReMask-Retry 的 PPL 改善实为退化假象）共同表明，DLM 的 training-free 推理时计算扩展面临根本性困难。"

### 3.2 与 PPL-Diversity 发现结合形成强叙事

将 ReMask-Retry（0.6B）的"PPL 改善 = 退化假象"发现与 Dream-7B 的"PPL 无改善"发现结合，形成了一个统一的负面叙事：

- 小模型：推理时修正降低 PPL，但通过退化实现——不是真实质量改善
- 大模型：推理时修正不降低 PPL，质量不退化但也不改善
- 结论：**当前 training-free 重遮蔽方法在 DLM 上存在效果天花板**

这种"小模型假阳性 + 大模型真阴性"的发现模式非常有说服力，且对社区极有警示价值。

### 3.3 适合的发表渠道

- **EMNLP Findings**：负面结果 + 方法论反思（"PPL 作为 DLM 质量指标的可靠性"）
- **ACL Workshop on Negative Results**：专门接收重要负面发现
- **ARR Rolling Review**：灵活时间线，可等 IGIR 结果后再决定最终叙事

### 3.4 节省社区资源的价值

DLM 推理时改进是 2025-2026 的热门方向（ReMDM、Prism、CoRe、Self-Rewarding SMC 等）。一篇严谨的负面结果论文可以帮助后来者避免重复走我们走过的弯路，特别是：

- 避免在小样本上过度解读 PPL 改善
- 避免忽视 PPL 与退化的关联
- 建立"必须多 seed + PPL/diversity 双指标"的评测规范

## 四、更有前景的转向方向

### 4.1 从 PPL 转向 benchmark 准确率（最高优先级）

18 轮迭代最大的教训是 PPL 不可信。但这并不意味着推理时修正没有价值——只是我们一直在用错误的指标衡量。

**具体方案**: 在 GSM8K（数学推理）和 HumanEval（代码生成）上直接测量准确率。这些任务有明确的对错判断，不受退化干扰。Dream-7B 在这些 benchmark 上已有公开基线数字，可以直接对比。

如果推理时修正能将 GSM8K 准确率从 X% 提升到 X+2%，即使 PPL 不变，这也是一个有意义的结果。

### 4.2 IGIR 的 MH 接受机制（值得一试）

如上文分析，IGIR 的核心创新（MH 接受 + 多轮迭代 + 复合质量评分）在概念上解决了当前方法的真实失败模式。建议先实现 IGIR-Lite（~300 行代码，1.1x 开销），在 100 题 GSM8K 子集上快速验证。这需要 3-5 天，是投入产出比最高的下一步。

### 4.3 条件式推理时修正（有潜力）

当前结果暗示推理时修正可能对特定类型的 prompt 有效（高 PPL 尾部、事实性知识问答）。如果这一假设成立，可以设计一个**条件式修正策略**：

1. 先生成初始输出
2. 用轻量指标（如生成过程中的平均置信度）判断是否需要修正
3. 仅对低置信度生成执行重遮蔽修正

这比"无差别修正所有生成"更高效，也更可能产生显著效果。从 full-scale 数据看，TCR 在约 51.5% 的 prompt 上改善了 PPL——如果能提前识别这 51.5%，仅对它们施加修正，效果可能更好。

### 4.4 跨模型系统性基准测试（稳健发表路径）

在统一实验框架下，系统对比 ReMDM（4 种策略）、TCR、熵引导、温度退火、Best-of-N 在 Dream-7B 和 LLaDA-8B 上的表现。这填补了文献调研中识别的"空白 1"（缺乏统一对比），即使所有方法都无效，一篇组织良好的 benchmark 论文也能被 EMNLP 接收。

### 4.5 理论分析：为什么 training-free 重遮蔽在 DLM 上困难

利用 IGIR 提案中的理论工具（IB 框架、最优重遮蔽比例闭式界），做一篇分析论文：

- 计算 Dream-7B 的校准误差 epsilon_cal
- 用闭式界预测最优重遮蔽比例
- 验证理论预测与实验结果的一致性
- 分析"DLM 推理时扩展的理论瓶颈在哪里"

这条路径不依赖于实验方法的成功，纯理论 + 验证性实验即可成文。

## 五、总结：乐观的底线

虽然 full-scale 结果令人失望，但从乐观角度看：

1. **我们没有得到虚假的正面结果**——这比发表一篇后来被推翻的论文要好得多
2. **PPL-Diversity 发现**是 18 轮迭代中最有价值的成果，可以独立发表
3. **IGIR 尚未测试**，其 MH 接受机制和准确率导向的评测方式代表了根本不同的方法论
4. **Dream-7B 不退化**的发现为更复杂方法留下了空间
5. **TCR 压制尾部异常值**的信号值得深挖，可能是一篇子集分析的种子
6. **社区需要严谨的负面结果**——这本身就是重要贡献

**推荐行动优先级**:
1. 先做 TCR 的子集分析（2 小时），确认或排除"选择性修复差生成"假设
2. 实现 IGIR-Lite 并在 GSM8K 100 题子集上验证（3-5 天）
3. 根据 IGIR 结果决定论文方向（methods paper vs negative results paper vs benchmark paper）
