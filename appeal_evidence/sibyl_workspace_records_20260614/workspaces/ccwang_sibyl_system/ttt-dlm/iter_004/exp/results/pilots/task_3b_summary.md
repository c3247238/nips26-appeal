# Task 3b Pilot 总结：Self-Contradiction Probing (SCP)

## 结果概要

| 指标 | Vanilla | SCP | 差异 |
|------|---------|-----|------|
| 准确率 | 12.5% (2/16) | 6.25% (1/16) | -6.25pp |
| distinct-2 | 0.838 | 0.885 | +0.047 |
| rep-3 | 0.101 | 0.058 | -0.043 (改善) |
| 平均时间/样本 | 8.0s | 52.5s | 6.5x 开销 |

## SCP 诊断统计

- **自矛盾检测率**: 1.9%（远低于预期 5%-50% 范围）
- 总探测 token 数: 21,114
- 总发现矛盾数: 281
- 总重遮蔽 token 数: 281
- 每样本约探测 10 次（步骤间隔=8，warmup=25%，stop=85%）

## Pass Criteria 评估

| 条件 | 结果 | 说明 |
|------|------|------|
| SCP 无错误运行 | PASS | 16 样本全部成功生成 |
| 矛盾率在 [5%, 50%] | FAIL | 仅 1.9%，模型对已揭示 token 的预测高度一致 |
| SCP acc >= vanilla | FAIL | 6.25% < 12.5% |
| distinct-2 >= 0.7 | PASS | 0.885 |

**整体判定: CONDITIONAL-GO**

## 分析

1. **矛盾率过低 (1.9%)**：Dream-7B 对已揭示 token 的 leave-one-out 预测高度一致，说明模型在局部上下文中的自洽性很强。这意味着 SCP 的假设——"通过检测自矛盾 token 来发现错误"——在该模型上信号太弱，无法有效识别全局层面的错误。

2. **准确率下降**：SCP 的重遮蔽干扰了正常的去噪收敛过程。虽然只有 1.9% 的 token 被重遮蔽，但这些扰动可能在后续去噪步中累积传播。

3. **文本多样性改善**：SCP 的 distinct-2 略高、rep-3 略低，说明重遮蔽在一定程度上打破了重复模式，但这并未转化为准确率提升。

4. **计算开销过大 (6.5x)**：每次 SCP 探测需要对所有已揭示 token 做 leave-one-out 前向传播，即使有 batch 处理，开销仍然显著。

## 对信息增强谱系的启示

- Level 1 (DMI): 0% 准确率，1.2x 开销 — 轻量但有害
- Level 2 (SCP): 6.25% 准确率，6.5x 开销 — 开销大但信号不足
- 两个低层级消融基线均未能改善 vanilla，这强化了 DTA (Level 3) 在参数空间做优化的必要性——token 级和 embedding 级的跨步信息传递在 Dream-7B 上效果有限。
