## 一、优点

### 1.1 信息论上界（Proposal 3：Delta_I 分析）是整个项目最重要的理论贡献

理论家提出的"信息孤岛差距" Delta_I(t) = I(Y_t; h_{T:t+1} | x_t, X_{R_t}) 是一个根本性的正确问题：**在投入工程资源修复 DaL 之前，我们必须先知道跨步记忆在原则上能提供多少信息**。如果 Delta_I 很小，那么无论修复什么工程问题，改进上限都是微不足道的。这个诊断（D0d）比 D0c（SSL-任务相关性）更为根本——它先于 D0c 发生，是所有后续投资的前提。

反对者强烈认同理论家将 Proposal 3 列为最高优先级。这是项目应该在第一次迭代就做的分析，但没有做。亡羊补牢，现在做也比不做好。

### 1.2 Lyapunov 稳定性分析（Proposal 5）对 P3 失败的诊断合理

"gate 作为增益参数，存在最大稳定值 beta_max"的分析是对 P3 失败的一个正确的工程-理论解释：gate=0.007 远低于 beta_max，所以系统是稳定的，但也是无效的——这是过度保守导致的性能下限，而非工程失败。这个分析给出了 gate repair 的理论指导（beta_optimal ≈ beta_max/sqrt(2)），而不只是"把 sigmoid(-5) 改成 sigmoid(-2)"的经验直觉。

### 1.3 对"现有理论不覆盖 DaL 场景"的诚实承认

理论家明确指出"现有 DLM 收敛理论都假设固定的 denoiser，TTT 引入了非平稳的反向过程，超出了所有现有框架"。这种诚实的理论定位（"我们在理论空白中工作"）比声称已有理论支持 DaL 要科学得多。反对者承认，从理论视角来说，理论家在这里比其他视角更诚实。

## 二、批评

### 2.1 定理草稿（Theorem Sketches）的证明义务未被严肃对待

理论家提出了五个"定理草稿"，但其中大多数面临严重的假设问题，而理论家只在"失败模式"中轻描淡写地提到：

**关于 Theorem 1（OCO 遗憾界）**：
- 假设 l_t 是 L-Lipschitz 和 beta-smooth 的。在 Transformer hidden states 上这几乎肯定不成立——注意力机制产生尖锐的 loss 景观（sharp loss landscape），注意力分数的不连续性违反了 L-Lipschitz 条件。
- "单调精化结构"的假设（R_0 ⊂ R_1 ⊂ ... ⊂ R_T）在 ReMDM 类的重掩码（remasking）策略中被明确违反。新的 baseline_remdm pilot（ReMDM -9pp）表明，这种重掩码策略在实践中是有害的——这个反例不仅挑战了实践价值，也挑战了理论假设。

**关于 Theorem 4（信息论界）**：
- 数据处理不等式界 Delta_I(t) <= I(h_{T:t+1}; X_{M_t} | X_{R_t}) 需要估算右侧项。理论家提出用变分信息瓶颈（probe network）来估算，但承认"probe network 可能提取不出全部信息，导致界过松"。一个过松的上界没有任何实际价值——如果测量出来 Delta_I < 0.1 bits/token，这可能是界本身太松，而不是信息真的少。理论家需要更严格地论证 probe network 方案能够给出足够紧的界，否则整个实验计划是建立在沙地上的。

### 2.2 理论框架在新 pilot 证据下未做充分更新

alt_a_pilot 的 +5pp（info-gain soft）是整个项目迄今为止最强的正向任务级信号。理论家的整个框架是围绕"DaL 是否有理论支撑"构建的，但没有提供 alt_a pilot 成功机制的任何理论解释。

**关键问题**：信息增益优先解码为什么有效？理论家拥有分析这个问题的最好工具：
- 信息增益解码是否等价于最大化 Delta_I(t)？（如果是，这将直接连接 Proposal 3 的分析和 alt_a 的实验成功）
- 信息增益解码是否对应于理论家 Proposal 2 中提到的 FIM 最优方向？
- 信息增益目标是否解决了 SSL-任务对齐问题——即，信息增益损失的 FIM 方向是否与任务损失的 FIM 方向更对齐？

这些问题的答案可能将 alt_a 的工程成功转化为理论理解，产生远比"DaL 遗憾界"更重要的贡献。理论家错过了这个机会。

### 2.3 Proposal 2（K-FAC TTT）的实践优先级被高估

理论家将 K-FAC TTT 列为"第三优先级"的改进方案。在当前阶段（项目还没有任何正向任务级结果），花费工程资源实现 K-FAC 是一个严重的优先级错误。

原因：
1. K-FAC 只有在 DaL 基本有效的前提下才有意义——它是"优化已经工作的系统"，而不是"修复不工作的系统"。
2. Delta_I（Proposal 3）才是决定是否继续投资 DaL 的先决条件。如果 Delta_I < 0.1，K-FAC 再好也没有意义。
3. alt_a 已经通过训练无关方法取得了 +5pp——在这个证据下，花费工程资源在 K-FAC TTT 上的机会成本极高。

反对者强烈建议重新排序：Proposal 3 -> Proposal 1+5（理论框架，低成本纯理论）-> 扩展 alt_a 分析，K-FAC 推迟到有正向实验信号后再考虑。

### 2.4 表达性分离定理（Theorem 5，Proposal 4）的实验检验设计缺乏说服力

理论家提出"TTT-MLP vs GRU 的性能差距应该在推理步骤更多时更大"作为表达性分离的实验检验。这个设计有严重缺陷：

**问题 1**: MetaState-GRU pilot 已经显示 GRU = 43.75% vs vanilla 50.0%（n=16，-6.25pp）。如果连 GRU baseline 都低于 vanilla，那么 GRU vs TTT-MLP 的比较研究就失去了最基本的意义——我们比较的是两个都比 vanilla 差的方法。

**问题 2**: "推理步骤越多，差距越大"的预测假设 GRU 和 TTT-MLP 至少一个比 vanilla 好。如果两个都比 vanilla 差（这是目前最可能的情况），那么"差距随复杂度增加"可能只是反映两个方法随复杂度增加而退化的速率不同，而不是表达性的优势。

反对者建议：在任何 TTT-MLP vs GRU 的比较研究之前，必须先有一个至少比 vanilla 好的基准。否则这个研究是在探索"哪个负面方法更不差"，不具有发表价值。

**分项评分**:

## 四、核心建议

**建议 1（最高优先级）**: 立即执行 D0d（Delta_I 估算），这应该是所有其他实验的前置条件。使用 probe network 估算 Delta_I，但同时需要设计对照实验验证 probe 的信息提取质量（否则结果无法解释）。如果 Delta_I < 0.1 bits/token，整个项目应该立即全面转向 alt_a 系列方法。

**建议 2（高优先级）**: 将理论工具（FIM 方向分析、信息增益分析）应用于解释 alt_a_pilot 的成功机制，而不是只用于分析 DaL 的失败机制。信息增益解码与 Delta_I 框架之间存在深刻的理论联系，挖掘这个联系是高价值的理论工作。

**建议 3（中优先级）**: 在所有"定理草稿"中，明确区分哪些是"有严格证明路径的猜想"，哪些是"启发性类比"。当前文档中这两类混在一起，可能误导读者（包括审稿人）认为这些是已证明的结果。

**建议 4（低优先级）**: 推迟 K-FAC TTT 直到有至少一个配置的 DaL 方法实现 > vanilla 的任务精度。在那之前，K-FAC 是错误的投资方向。
