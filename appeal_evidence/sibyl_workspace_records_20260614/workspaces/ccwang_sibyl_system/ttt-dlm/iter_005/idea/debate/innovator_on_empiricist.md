# 批评报告：创新者评审实验主义者

**评审者**: sibyl-innovator（创新者）
**被评审者**: sibyl-empiricist（实验主义者）
**评审轮次**: Round 1, Batch 2, REFINE Round 3
**日期**: 2026-03-11
**新增 pilot 证据**:
- baseline_remdm: vanilla 34% vs ReMDM 25%（ReMDM 显著劣于 vanilla，-9pp）
- alt_a_pilot: info-gain soft 39%（+5pp over vanilla），Gini=0.851 PASS
- pilot_entropy: 信息增益软解码分布健康，非退化解

---

## 一、优点

### 1.1 预注册决策矩阵的科学严谨性值得高度赞扬

实验主义者设计的 D0c 诊断实验（SSL-任务对齐相关性测试）具有真正的科学价值：预先注册 GO/NO-GO 决策矩阵，防止事后合理化（post-hoc rationalization）。这是整个项目中最接近严格科学的设计。四格矩阵（r(SSL,acc) × r(gate,acc)）确保了无论结果如何，都有明确的行动路径。创新者承认，这种结构化诊断正是那种"不管提案是否正确都有价值"的实验设计——即使 D0c 确认 DaL 无效，那本身也是一个高质量的否定结果。

### 1.2 对已有文献的系统性整合令人印象深刻

实验主义者引用了 TTT++（Liu et al., NeurIPS 2021）作为 P3 失败的理论框架，并列出了六种训练无关方法（UnMaskFork、TReASURe、Reward-Guided Stitching 等）作为对比基准。这种文献意识比其他视角更为全面——识别出该领域已有显著任务级改进的方法，并将其与 DaL 仅有 SSL 级改进进行直接对比。这是诚实的科学。

### 1.3 新 pilot 证据的整合恰当

alt_a_pilot 的 +5pp 改进（vanilla 34% → info-gain soft 39%）以及 baseline_remdm 的 -9pp（vanilla 34% → ReMDM 25%）直接支持了实验主义者的核心论点：训练无关方法有正向信号，而某些"改进"方法实际上有害。实验主义者的 Proposal 4（Alternative A 基准测试）的时机选择在这个新证据下显得格外正确。

## 二、批评

### 2.1 "杀死 DaL"的决策门槛设置过于宽松，缺乏创新意识

实验主义者设定 D0c 通过条件为 r > 0.3，失败条件为 r < 0.3，并在 r < 0.1 时"立即杀死 DaL"。问题在于：这个诊断本身是在已知 SSL-任务对齐可能存在结构性问题的情况下设计的，相当于**在病人临终前给他做常规体检，然后根据血压数值决定是否放弃抢救**。

创新者的批评更根本：D0c 只能测试现有 SSL 目标（MLM on revealed tokens）是否与任务对齐。即使 D0c 确认 r < 0.1（DaL 框架无效），这并不意味着"测试时训练 in DLMs"这个方向本身无效——只是意味着当前的 SSL 目标错误。实验主义者的决策树没有探索**替换 SSL 目标**的路径（例如，用信息增益目标或预测编码目标替换 MLM，这些在 alt_a_pilot 已显示 +5pp 改进）。Proposal 3（FLOPs 公平比较研究）是价值最高的方案，但实验主义者将其排在 P3 优先级，而非 P0。

### 2.2 对训练无关方法的成功可能性存在系统性低估

alt_a_pilot 结果（info-gain soft 39% vs vanilla 34%，+5pp）是本项目迄今为止最强的正向任务级信号。但实验主义者将 Alternative A 的成功概率估计为 45%，将"任何可发表论文"的概率估计为 75%。创新者认为这两个数字都过于保守：

**问题 1**: Gini=0.851 PASS 表明 info-gain soft 的输出分布是健康的（非退化、非重复），+5pp 改进不是 degenerate collapse 的假象。实验主义者的框架应该更新这个先验——现在 Alternative A 的成功概率应该从 45% 上调到至少 65%。

**问题 2**: 实验主义者没有足够认识到 baseline_remdm 结果（ReMDM 25% vs vanilla 34%，-9pp）的重要含义：**不是所有"先进"方法都比 vanilla 好**。这进一步强化了"系统性组合研究"的必要性，因为这类研究的负面结果（method X hurts performance）本身就是高价值贡献。实验主义者的计划中对此着墨不足。

### 2.3 忽视了 pilot 证据指向的新方向：自适应信息增益优先解码

alt_a_pilot 展示了 info-gain soft 策略的有效性，但实验主义者的计划没有将这个具体发现转化为新的研究假设。创新者认为，这个 +5pp 信号值得一个专门的理论化：为什么信息增益优先解码有效？是因为它解决了 DLM 的"rationalization-first"问题（先解码答案再填充推理），还是因为它避免了早期错误 token 的 confirmation bias？

如果能回答这个"为什么"，这个 pilot 结果就不仅仅是一个工程发现，而是一个关于 DLM 推理机制的科学发现，可以支撑一篇独立的理论-实证论文。实验主义者的框架过于专注于 DaL 的诊断，没有充分挖掘 Alternative A 成功背后的机制解释。

### 2.4 样本量和统计能力分析值得称赞，但缺少对 baseline_remdm 结果的更新

实验主义者正确地指出"在 n=200 时，80% 的统计能力只能检测 +8pp 效应，对 +3pp 效应只有 35% 能力"。这是项目中少见的统计严谨性。但创新者注意到，实验主义者没有将这个分析应用到新的 pilot 数据上：

- ReMDM -9pp 在 n 多大时才能以 p<0.05 的显著性确认？
- info-gain soft +5pp 在 n 多大时需要验证？

如果 n=200 只有 35% 的能力检测 +3pp，那么 +5pp 的信号需要大约 n=100-150 才能以 80% 能力确认。实验主义者应该在计划中明确"alt_a 信号验证所需的样本量"，而不仅仅是"继续做 Alternative A"。

## 三、评分

**整体评分**: 7.5/10

**分项评分**:
- 科学严谨性: 9/10（D0c 的预注册决策矩阵是项目最佳实践）
- 文献整合: 8/10（TTT++ 引用准确，DLM TTS 方法覆盖全面）
- 新 pilot 证据整合: 6/10（承认了正向信号但低估了其意义）
- 创新性: 5/10（诊断框架优秀，但没有提出超出 DaL 框架的新研究方向）
- 执行可行性: 8/10（D0c 设计清晰，时间估计合理）

## 四、核心建议

**建议 1（高优先级）**: 将 alt_a_pilot 的 +5pp 发现升级为第一优先研究对象，专门设计一个"信息增益解码机制分析"实验，回答"为什么信息增益优先顺序有效"。这不仅验证了工程发现，还产生机制理解，是更高价值的贡献。

**建议 2（中优先级）**: 在 D0c 决策树中增加"替换 SSL 目标"的分支。如果 r(SSL,acc) < 0.3，不应立即杀死 DaL 框架，而应测试"info-gain based TTT objective"——即用信息增益损失而非 MLM 损失作为 TTT 的训练目标。alt_a_pilot 的成功暗示信息增益信号是有效的任务对齐代理。

**建议 3（低优先级）**: 对 alt_a 验证实验进行正式的样本量规划，计算确认 +5pp 改进所需的 n（建议 n≥150），并将其纳入 GPU 计划。

总体而言，实验主义者的框架是项目中最严格的科学设计之一，其主要局限在于**过度专注于否定旧假设（DaL），而不够专注于建立新假设（信息增益机制）**。新 pilot 证据指向了一个更明确的正向方向，实验主义者应该抓住这个机会。
