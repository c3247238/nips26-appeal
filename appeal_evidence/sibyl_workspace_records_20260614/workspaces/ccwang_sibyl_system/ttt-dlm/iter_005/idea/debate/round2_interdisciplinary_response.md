## 一、对实验主义者批评的回应

### 1.1 对批评 1 的回应：「对 alt_a_pilot +5pp 信号缺乏反应，整体框架仍以 DaL 修复为中心」

实验主义者提出了跨学科者视角最重要的批评：四个类比框架（Turbo、预测编码、模拟退火、亲和力成熟）全部聚焦于「如何修复 DaL」，完全没有讨论「信息增益解码顺序为何有效」。实验主义者进一步指出，信息增益解掩蔽与三个类比框架都有深度对应：Turbo 的外生信息最大化、主动推断的自由能最小化、退火的最优步骤。

**跨学科者完全接受这个批评，且承认这是最重要的盲点**。

原始文档的确在「如何诊断和修复 DaL」上耗费了 23 GPU 小时的实验计划，而对 alt_a_pilot 的成功仅给出了两段描述性评论（2.2 节），没有将其转化为系统的跨学科理论化。这是一个真正的框架性遗漏。

**现在补充这个分析**：

信息增益优先解码（Info-Gain Soft）的成功，从三个类比框架来看，指向一个统一的理论解释：

**Turbo 框架**：信息增益解掩蔽顺序在数学上等价于最大化每步向 TTT 层（或骨干网络）提供的外生信息量。正式地，在步骤 t，选择解掩蔽 token i* 使得：

```
i* = argmax_i I(X_i; X_masked | X_revealed_t)
```

这正是 Turbo 解码中「最大化外生信息转移」的贝叶斯形式化。Info-Gain Soft 让骨干网络在已有最多高置信度已揭示 token 的上下文中决定关键 token，这从 Turbo 框架看是「最大化每步解码中外生信息的贡献」。相比之下，随机解码顺序（vanilla）或 ReMDM 的随机重掩码，都是在提供低外生信息量的条件下做决策——这解释了为什么 Info-Gain Soft 成功而 ReMDM 失败。

**主动推断框架（自由能最小化）**：在 Friston 的自由能原则中，最优「行动」（action）是使期望自由能（expected free energy）最小化的选择。在 DLM 解码中，「行动」就是选择下一个解掩蔽的 token。信息增益解掩蔽顺序正是这个原则在 DLM 中的实现：选择使期望预测误差（未来步骤的困惑度）最小化的 token。这是自由能框架的直接预测：「最大化信息增益 = 最小化期望自由能」。

**模拟退火框架**：在退火的关键温度窗口（r ∈ [0.3, 0.7]，即 P2 中梯度 SNR 峰值区），信息增益选择对应最大减少系统「能量」（不确定性）的操作步骤。从 Gibbs 采样的角度，在相变临界点附近，优先解掩蔽对系统状态有最大确定性影响的 token，等价于选择「最能降低能量的旋转翻转」——这是统计物理中最优退火路径的定义。

**综合预测**：这三个框架对 Info-Gain Soft 为何有效给出了**相互独立且相互支持**的理论解释。这不仅为 alt_a_pilot 的成功提供了事后理论基础，更重要的是，它们共同预测了**进一步提升信息增益解码的方向**：

1. **自适应 tau 调度**：在临界区域（r ∈ [0.3, 0.7]）使用较低 tau（更确定性的选择），在临界区域外使用较高 tau（保持探索性）。这结合了 Greedy（-18pp）的失败教训（全局确定性过冷）和 Soft（+5pp）的成功。预测：自适应 tau 应比固定 tau=1.0 更好

2. **多步前瞻信息增益**：当前 Info-Gain Soft 是贪婪的（每步独立最大化当步信息增益）。Turbo 框架预测，考虑未来 k 步的「期望信息增益」会更优（类似 beam search 在 AR 模型中的改进）。预测：k=2 的前瞻信息增益比 k=1 的贪婪版本更好

3. **外生性验证**：如果 Info-Gain Soft 的成功来自最大化外生信息，那么在解码过程中测量「每步解码的条件互信息 I(X_new; X_masked | X_revealed)」应该显示 Info-Gain Soft 的每步信息增益显著高于 vanilla。这是一个可测量的、无需额外 GPU 时间的诊断（在已有的 alt_a_pilot checkpoint 上分析）

**优先级调整**：跨学科者将信息增益解码的跨学科理论化升级为**第一优先项**，在实验预算中分配 4 GPU 小时用于上述自适应 tau 实验（成本从 E_turbo 的余弦相似度诊断预算中调整）。DaL 修复实验降级为第二优先项，仅在 D0c 通过时继续。

### 1.2 对批评 2 的回应：「余弦相似度阈值（>0.7）缺乏数学推导」

实验主义者提出了一个精确的方法论批评：H_turbo 的可测量预测「TTT 输出与骨干输出余弦相似度 > 0.7」中，0.7 这个阈值没有从 Turbo 解码理论中推导出来。实验主义者进一步指出，余弦相似度测量的是向量方向重叠，而 EXIT 图表测量的是条件熵减少量——两者在高维空间中相关性不明确。

**跨学科者部分接受，但对指标替换方案保留意见**：

**接受部分**：0.7 阈值确实来自直觉而非正式推导。在 Turbo 解码的 EXIT 图表框架中，「外生信息量」是通过条件互信息（I_E）来量化的，而不是向量余弦相似度。余弦相似度与外生信息量之间没有精确的数学关系，这是一个真实的方法论弱点。

**保留意见 1**：实验主义者建议的替代指标「测量 TTT 输出对骨干输出的条件互信息」在高维空间中在计算上极其困难。精确的高维互信息估计（例如，d_model=2048 的 Transformer 隐藏层）在小样本（100-200 个问题）下是极不可靠的，需要大量的核密度估计或神经网络互信息估计器，本身引入了大量方差。这不是「更简单的指标」，而是一个需要独立验证的测量方法。

**保留意见 2**：实验主义者建议「训练轻量级探测器（probe）测量 TTT 是否包含骨干不知道的信息」。这是一个更实际的替代方案，跨学科者接受这个建议。具体协议：
- 在 P3 的 DaL checkpoint 上，训练两个轻量级线性探测器（Logistic Regression，无需 GPU 训练）：
  - Probe A：从 h_backbone 预测掩蔽 token 的真实词元（基准）
  - Probe B：从 h_ttt 预测相同掩蔽 token（测试 TTT 的额外信息量）
- 如果 Accuracy(Probe B) - Accuracy(Probe A) > 1%：TTT 包含骨干不知道的外生信息，E_extrinsic 有理论基础
- 如果差异 < 1%：TTT 只在内生地重编码骨干已知的信息，Turbo 框架的诊断成立

这个「轻量级 probe 测试」的计算成本比完整的 E_turbo 更低（约 +0.5 GPU 小时，主要是推理时间），且提供了比余弦相似度更直接的「外生信息量」测量。

**修订后的 H_turbo 预测**：不再使用「余弦相似度 > 0.7」作为可测量预测，而是使用：「Accuracy(Probe_TTT) - Accuracy(Probe_backbone) < 1%（TTT 不包含外生信息）」。这在理论上更严格，在计算上更实际。

0.7 的阈值被完全放弃。新的「外生性判据」是 probe 精度差异，而不是向量方向相似度。

### 1.3 对批评 3 的回应：「E_anneal 的消融层次化不足」

实验主义者指出 E_anneal 将两个独立改变（对数掩蔽调度 + Quench-then-Anneal 协议）放入同一个实验，导致无法分离各自贡献。建议拆分为：
- E_anneal_a：仅改变掩蔽调度（+1.5 GPU 小时）
- E_anneal_b：仅改变 TTT 阶段分配（+1.5 GPU 小时）

**跨学科者完全接受这个批评**，这是最干净的实验设计批评之一。

实验主义者的「每次只改变一个变量」原则是正确的。E_anneal 的设计确实将两个因变量混在一起，这是跨学科者在设计时过度追求「理论整体性」（希望测试「完整的退火方案」）导致的实验设计错误。统计上，这相当于忽略了一个潜在的交互效应——对数调度和 TTT 阶段分配可能存在超加性（synergistic）或次加性（antagonistic）的相互作用，而当前设计无法检测。

**具体更新**：E_anneal 拆分为：
- **E_anneal_a**（+1.5 GPU 小时）：仅将临界区域的掩蔽调度从线性改为对数调度（r ∈ [0.3, 0.7] 使用 r(t) = 0.7 × exp(-0.5t)，其余区域保持线性），全程 TTT 不变
- **E_anneal_b**（+1.5 GPU 小时）：前 60% 步骤关闭 TTT（fast forward），后 40% 步骤开启 TTT（精细退火），保持线性掩蔽调度
- **E_anneal_ab**（+1 GPU 小时，条件运行）：仅当 E_anneal_a 和 E_anneal_b 都显示正向信号时，测试两者的组合

总成本：3-4 GPU 小时（与原来相近），但提供了独立效果估计和交互效应检测。

从模拟退火理论来看，这个拆分也有理论意义：对数调度影响的是「退火速率」（系统探索节奏），而 Quench-then-Anneal 影响的是「TTT 学习窗口」（快速权重的有效区域）。两者在理论上是正交的，因此加性效应（没有交互）是先验最可能的假设，但仍需实验验证。

### 1.4 对批评 4 的回应：「'至少一个成功概率 87%' 的独立性假设错误」

实验主义者提出了精确的统计批评：87% 的计算假设四个修复方案的成功概率相互独立，但这四个方案都依赖「SSL 信号与任务准确率有某种程度的相关」这一共同前提，因此成功概率不独立——条件依赖 D0c 通过（25% 概率）。修正后的实际成功概率约为 0.25 × 87% + 0.75 × 5% ≈ 25%。

**跨学科者完全接受这个统计批评**，这是一个真实的概率计算错误。

实验主义者的修正是正确的。四个跨学科修复方案（E_extrinsic、E_restart、E_anneal、E_hier_ttt）在逻辑上都以「DaL 的基本机制（SSL → 任务性能）存在某种有效性」为前提——如果 D0c 显示 r < 0.1（SSL 与任务完全无关），则四个方案的成功概率**全部接近零**，不是各自独立的。

**修订后的概率陈述**：

- **无条件概率**（不知道 D0c 结果）：至少一个跨学科修复方案有效 ≈ **0.25 × 87% + 0.75 × 5% ≈ 25%**
- **条件概率**（已知 D0c 通过，r > 0.3）：至少一个跨学科修复方案有效 ≈ **87%**
- **条件概率**（已知 D0c 失败，r < 0.3）：至少一个跨学科修复方案有效 ≈ **5-10%**（只有 E_restart 和 E_anneal 可能在没有 SSL-任务相关的情况下仍有价值，但可能性很低）

这个修正有重要的优先级含义：跨学科修复方案的期望价值（25%）低于 Alternative A（alt_a_pilot 已显示 +5pp）的期望价值（约 55% 成功概率）。因此，在 D0c 结果出来之前，**Alternative A（信息增益解码系统研究）应该获得优先的 GPU 分配**，跨学科 DaL 修复实验的 23 GPU 小时应该降级为「D0c 通过后的条件计划」。

**优先级重排**：
1. **首先**（并行）：D0c 诊断（8-10 GPU 小时，含新增的余弦相似度诊断）+ Alt A 信息增益系统研究（自适应 tau，n=300，约 16 GPU 小时）
2. **如果 D0c 通过**：启动跨学科 DaL 修复实验（E_restart, E_anneal_a/b, E_extrinsic，按成本递增顺序）
3. **如果 D0c 失败**：全力推进 Alt A，跨学科框架转向「解释信息增益解码为何有效」而非「修复 DaL」

**修订的立场**：
- **最重要的修订**：将信息增益解码的跨学科理论化升级为**第一优先项**（之前完全被忽略），三个类比框架对 alt_a_pilot 成功的统一解释是现在框架的核心贡献
- **H_turbo 的可测量预测修订**：从「余弦相似度 > 0.7」改为「Probe_TTT 相对 Probe_backbone 的精度提升 < 1%（外生信息量低）」

- **87% 概率修订**：这是条件概率（以 D0c 通过为条件），无条件成功概率约 25%
- **优先级修订**：跨学科 DaL 修复实验降级为 D0c 通过后的条件计划，Alt A 信息增益研究优先级提升

**新增的立场**：
- 提出信息增益解码的三框架统一理论（Turbo、主动推断、退火），预测自适应 tau 调度和多步前瞻信息增益的改进方向
- 提出轻量级 probe 测试作为「外生性」的实际测量方法（替代不可靠的高维互信息估计）

## 三、共识点

经过实验主义者的批评，以下几点达成实质共识：

1. **alt_a_pilot 的成功需要跨学科理论化**：信息增益解码的 +5pp 不仅是工程发现，三个类比框架（Turbo、主动推断、退火）都自然导出「信息增益解掩蔽优于随机顺序」的预测。这是跨学科框架的最强正向贡献，需要系统研究

2. **外生性测量应用 probe 而非余弦相似度**：实验主义者的建议（轻量级探测器测量 TTT 是否提供骨干不知道的信息）是比余弦相似度更直接、更有理论对应的测量方法，跨学科者接受

3. **E_anneal 应拆分为独立消融**：每次只改变一个变量是实验设计的基本原则，跨学科者的「理论整体性」追求不能牺牲实验因果可解释性

4. **87% 概率是条件概率**：以 D0c 通过（25% 概率）为条件。无条件成功概率约 25%，低于 Alternative A 的预期成功率，因此 Alt A 优先是正确的资源配置决策

5. **自适应 tau 是下一步最有价值的 Alt A 实验**：从三个类比框架的预测来看，固定 tau=1.0 是次优的，临界区域的较低 tau 应该比均匀 tau=1.0 更好。这是跨学科框架对 Alt A 方向的新贡献
