## 一、对反对者四条核心批评的回应

### 回应批评1（最重要）：alt_a_pilot +5pp统计不显著，实用主义者轻描淡写

这是本轮辩论中最需要认真对待的批评。反对者明确指出：
> "The +5pp improvement is not statistically significant at n=100 (p=0.182)"
> "agreement分析：Info-Gain Soft获得了9个vanilla错误的题目正确，但失去了27个vanilla原本正确的题目"

**完全接受统计不显著的事实**：p=0.182在任何合理的显著性阈值下（p<0.05、p<0.10）都不能拒绝零假设。实用主义者在原提案中承认了A-CFG在n=16的不可信性，但对Info-Gain Soft在n=100的p=0.182赋予了过强信心，这确实是逻辑不一致。

**修正认识论立场**：alt_a_pilot的+5pp应该被重新定性为**弱探索性信号，而非已确认的正向效果**。它的正确使用场景是：
- 作为方向性指针（"这个方向值得验证"），而非已建立的事实
- 作为n=300验证实验的动机（"这个信号足够强，值得花2 GPU小时验证"），而不是EGF组合研究的直接基础

**EGF优先级的重新论证**：反对者质疑EGF的55%成功概率，因为其论据基础（各单独方法已验证有效）是薄弱的。实用主义者接受这个质疑，但提出更稳固的论据：

EGF的真正价值不在于"已验证方法的叠加"，而在于**系统性研究本身的方法论贡献**。即使A-CFG、Soft-Masking、Entropy-Adaptive Schedule各自都没有显著效果，一个系统研究三个方法如何相互作用（additive vs. sub-additive vs. conflicting）的论文，在现有DLM推理文献中仍然是有价值的——因为现有文献中没有这样的系统性研究。

**修正后的成功概率估计**：
- EGF：55% → 45%（因为基础方法的统计置信度低于原估计）
- 但EGF作为方法论贡献的发表价值不变（负面结果同样可发表）

### 回应批评2：CGPR成功概率35%被高估

部分接受。反对者列出了三个具体问题：

**问题1（Dream-7B置信度校准）**：接受。pilot_entropy显示双峰熵分布（大量接近0和接近5.0的位置），意味着置信度增量信号高度噪声化。这是CGPR面临的真实风险。

**问题2（与Soft-Masking重叠）**：接受。CGPR与Soft-Masking的核心差异仅在"置信度增量vs绝对置信度"，边际新颖度较低。如果Soft-Masking（IBM开源checkpoint）已经验证过，CGPR需要更清晰地论证其差异性贡献。

**实用主义者的立场**：不放弃CGPR，但降低其优先级，并修改实验设计：
- 在CGPR pilot之前，先运行Soft-Masking baseline（IBM checkpoint直接部署，<2 GPU小时）
- 如果Soft-Masking已经达到+3%以上，CGPR作为消融实验（仅测试"增量vs绝对"的差异）
- 如果Soft-Masking效果弱，CGPR作为独立候选方向运行

**修正后的CGPR成功概率**：35% → 28%

**问题3（alpha超参数敏感性）**：部分接受。15 GPU小时的grid search确实是计划中最大的单项计算消耗。修正：将alpha grid search从5个值（0.1, 0.3, 0.5, 0.7, 0.9）缩减到3个关键值（0.1, 0.5, 0.9），节约9 GPU小时，牺牲最优alpha的精度，换取更快的决策。

### 回应批评3：DaL门修复训练应严格先于D0c诊断

**完全接受**。反对者的批评是正确的：实用主义者在原计划中让Day 1 GPU 0同时开始门修复训练，这违反了"先诊断，后治疗"的原则。P3的教训正是"SSL损失下降不等于任务准确率提升"，而D0c诊断（SSL-任务对齐）的目的就是在门修复之前确认治疗方向是否正确。

**修正后的执行计划**：
```
Day 1：
  GPU 0: D0c诊断（minimal版本，4个配置，~2 GPU小时）
  GPU 1: A-CFG baseline（EGF Phase 1）
  GPU 2: RCR + Vanilla baselines（EGF Phase 1）
  GPU 3: COBA grid search D轴

Day 2（在D0c结果基础上决策）：
  如果D0c PASS（r > 0.3）：GPU 0启动门修复训练
  如果D0c FAIL（r < 0.3）：GPU 0转向SPD pilot或ADM pilot（来自创新者提案）
  GPU 1: SM + A-CFG composition（EGF Phase 2）
  GPU 2: CGPR pilot（Soft-Masking baseline先行）
  GPU 3: COBA W轴 + M轴扩展
```

这个修改使Day 1损失了8-10 GPU小时的门修复训练时间，但避免了"基于错误假设投入10 GPU小时"的风险。值得。

### 回应批评4：EGF组合研究的"加法性"假设缺乏实证支撑

部分接受，但对反对者的批评进行精确化。

反对者指出A-CFG和Entropy-Adaptive Schedule都在控制解掩蔽时机，可能存在冲突。这个技术观察是正确的。但反对者随后得出的结论——"如果发现主要方法对之间是冲突的，论文影响力大幅下降"——实用主义者不同意。

**实用主义者的回应**：发现"方法冲突"本身就是一个重要的研究发现。如果A-CFG和Entropy-Adaptive Schedule在实验中表现出sub-additive效果（甚至冲突），论文的贡献从"最优组合"转变为"方法冲突的系统性分析"——这仍然是有价值的，因为现有文献都假设这些方法可以无冲突地组合，没有人实验性地检验过这个假设。

**修正**：在EGF的论文定位上，从"证明组合优于单个"改为"系统性研究DLM推理方法的组合行为"。这个定位对正负结果都有同等的发表价值，降低了对特定实验结果的依赖。

## 二、对理论家四条核心批评的回应

### 回应理论家批评1：DaL门修复的"最低悬挂果实"判断需要修正

接受理论家的核心批评，并接受其建议。

理论家指出：beta_max = 1/(L_W * eta * G_max)，而sigmoid(-2) = 0.12的启发式选择可能不在稳定范围内。在门修复之前，先进行2 GPU小时的beta_max估算实验（通过梯度范数分析）。

**但有一点需要澄清**：理论家批评"在没有Delta_I估算的情况下，门控修复是修复症状而不知病因"。实用主义者的立场是，D0c（SSL-任务对齐）和门修复形成了合理的两步决策：
- D0c回答"SSL信号是否与任务对齐"（先做）
- 门修复回答"如果对齐，工程上如何确保信号传入骨干"（后做）

Delta_I（D0d）回答的是更根本的问题："跨步记忆是否有信息论意义上的改进空间"——这个问题的优先级高于D0c，实用主义者原计划中缺少这一步是真正的疏漏。

**修正**：将D0d（Delta_I估算）加入Day 1计划，与D0c并行运行：
```
Day 1 GPU 0: D0c minimal（4配置，~2 GPU小时）
Day 1 并行（CPU/GPU 0后）: D0d诊断（probe网络估算Delta_I，~4-6 GPU小时，可与D0c串行）
```
如果D0d显示Delta_I < 0.1，即使D0c PASS，门修复也没有意义，整个DaL路线应立即终止。

### 回应理论家批评2：EGF组合框架缺乏理论兼容性预分析

接受并将其整合到EGF实验设计中。

理论家识别出两个关键冲突点：
1. A-CFG（低置信度重掩码）和RCR（运行置信度重掩码）在相同决策点竞争
2. Soft-Masking的连续embedding与A-CFG的二进制重掩码决策之间的信息不一致

**实用主义者的处理方案**：不是"在理论层面解决冲突"，而是**把冲突检测纳入实验设计本身**：
- Phase 1（Day 1）：独立运行每个方法，建立baseline
- Phase 2 (a)（Day 2）：运行正交对（A-CFG+SM：应无冲突，embedding轴和guidance轴分离）和可能冲突对（A-CFG+RCR：都操作mask模式）
- Phase 2 (b)：通过实验结果判断是否存在冲突（sub-additive = 冲突指标）
- 如果发现冲突：这是论文的发现，不是失败

理论家的建议"在执行前进行理论兼容性分析"是理想的，但在7-10天的时间窗口内，实验性检测比理论预分析更实用。两者可以结合：理论家的冲突预测（"A-CFG和Entropy-Adaptive会冲突"，"SM和A-CFG应该兼容"）作为实验假设，实验结果验证或否定这些预测。

### 回应理论家批评3：COBA发表概率65%过于乐观

部分接受，但不接受下调到50%。

理论家的两个风险点：
- **风险1**：对数线性模型quality(D,W) = alpha*log(D) + beta*log(W) + gamma可能对不同任务有不同函数形式（非对数线性）
- **风险2**：W轴的Best-of-N选择机制功率分析缺失

**对风险1的回应**：非对数线性关系不是失败，而是发现。如果D轴有非单调效应（ReMDM -9pp已经暗示），那么"DLM推理的D轴扩展存在最优点而非单调增益"本身就是重要发现。论文可以从"最优D×W分配研究"重新定位为"DLM推理扩展的Scaling Law研究"，后者的影响力可能更大。

**对风险2的回应**：接受。将功率分析加入COBA的实验设计：
- 对于W=16，n=200的GSM8K，majority vote的置信区间约±3.5%（基于二项分布）
- 这意味着至少需要看到±4%的真实效果才能在W轴检测到显著差异
- 实验设计调整：将W的评估样本从200增加到500，使置信区间收窄到±2.2%

修正后COBA成功概率：65% → 60%（主要因为分析不确定性，不是数据收集不确定性）

### 回应理论家批评4：计划对alt_a_pilot正向信号响应不足

接受，并结合反对者的批评一起更新计划。

理论家建议的Day 1计划更新（alt_a扩展验证优先）与反对者对+5pp统计不显著性的批评之间存在张力：理论家说"信号更强的方向应该获得更多GPU"，反对者说"这个信号统计上还不显著"。

**实用主义者的综合立场**：
- 正确的做法是**同时运行验证和继续探索**，而非二选一
- alt_a扩展（n=300）只需要~2 GPU小时，不影响其他GPU的计划
- 但不应该因为一个p=0.182的信号就把所有资源都转向alt_a

**修正后的Day 1 GPU分配**：
```
GPU 0: D0c minimal + D0d Delta_I诊断（串行，~2+4=6 GPU小时）
GPU 1: A-CFG baseline + alt_a n=300扩展验证（并行运行，共6小时）
GPU 2: RCR + Soft-Masking baselines
GPU 3: COBA D轴网格搜索（D∈{32,64,128,256,512}，W=1）
```

这个分配在Day 1就产出：D0c/D0d结果（DaL方向决策基础）、A-CFG独立验证、alt_a统计显著性确认/否定、COBA D轴初步数据。

## 四、共识点

与反对者的共识：
1. alt_a_pilot +5pp在统计上不显著（p=0.182），不能作为EGF优先级的主要论据
2. D0c诊断（SSL-任务对齐）必须严格先于DaL门修复训练——这违背了原计划，现已修正
3. CGPR与Soft-Masking的边际差异有限，应先测Soft-Masking baseline再决定CGPR
4. COBA是项目portfolio中期望价值最高的单一方向（两者在这点完全一致）

与理论家的共识：
1. COBA的对数线性模型需要备用分析策略（非单调性检验）——已列入修正计划
2. CGPR的置信度delta机制与alt_a的信息增益策略在理论上是近似的，可以考虑合并成"动态信息引导解码框架"
3. 方法兼容性分析（理论家）和实验性冲突检测（实用主义者）可以结合，前者提供预测假设，后者提供实验验证
4. D0d（Delta_I估算）是所有DaL路线投资的先决条件，应该与D0c并行运行
