# 批评报告：跨学科者评审反对者

**评审者**: sibyl-interdisciplinary（跨学科者）
**被评审者**: sibyl-contrarian（反对者）
**评审轮次**: Round 1, Batch 2, REFINE Round 3
**日期**: 2026-03-11
**新增 pilot 证据**:
- baseline_remdm: vanilla 34% vs ReMDM 25%（ReMDM -9pp，重掩码策略有害）
- alt_a_pilot: info-gain soft 39%（+5pp），Gini=0.851 PASS
- pilot_entropy: 输出分布健康，非退化

---

## 一、优点

### 1.1 反对者的三个挑战假设在新 pilot 证据下获得了强力支持

跨学科者在审阅之初就注意到：新的 pilot 数据是反对者三个核心论点的强力验证。特别是：

**C3（更多推理时计算不可靠地改善 DLM 推理）的直接验证**: ReMDM -9pp 是对这个假设最强的单点证据——ReMDM 代表"更多重掩码步骤 = 更多计算"的方向，但结果不仅没有改善，而且造成了显著的 -9pp 退化。在跨学科者的统计物理框架中，这对应于"过度退火"（over-annealing）：ReMDM 在已经收敛的区域继续施加随机扰动，破坏了系统已经达到的有序态，使其重新陷入无序。这是一个 DLM"临界温度"之外的区域进行过度扰动的典型案例，从物理上提供了为什么 ReMDM 有害的机制解释。

**alt_a_pilot +5pp 与反对者 C2 假设的联系**: 反对者认为 SSL 目标（MLM on revealed tokens）与任务性能存在结构性解耦，建议转向"训练无关方法或任务对齐奖励信号"。info-gain soft 的成功正是一种任务对齐的训练无关策略——它不依赖任何 SSL 代理损失，而是直接优化每步解码的信息量，这是一种隐式的任务对齐（信息量最大化与回答问题的能力自然对齐）。反对者应该将 alt_a 的成功视为 C2 批判的实验支持，而不仅仅是"训练无关方法有正向信号"。

### 1.2 "Pareto 前沿由 vanilla + 训练无关引导方法主导"的预测令人信服

反对者提出假设 C3："对于 7B 规模的 DLM 推理任务，accuracy-vs-FLOPs 的 Pareto 前沿由两种方法主导：(1) vanilla 密集去噪与步骤加倍，(2) 训练无关引导方法（A-CFG、CoRe）"。跨学科者认为这是项目历史上最精准的预测性假设之一，而 alt_a_pilot 的结果是对其的部分实证支持。

从统计物理角度，这个预测有理论基础：在临界区域（mask ratio ≈ 0.6）以外，系统的优化格局由能量景观的整体结构决定，而非单步更新的细节。训练无关方法通过改变"哪些 token 先解码"（解码顺序 = 退火路径）来影响最终状态，这等价于在相同计算预算下探索不同的退火路径——某些路径（信息增益优先）更能避开局部极小，从而以相同或更少的步骤达到更好的状态。

### 1.3 对自身偏差的元认知分析令人印象深刻

反对者明确承认"22 次迭代的负面结果可能产生了锚定偏差（30% 概率）"，并将此作为自己立场的一个已知风险。跨学科者注意到，这种元认知能力在五个视角中是最成熟的。相比之下，创新者的"新颖性偏好"和实用主义者的"工程可行性偏好"都是未被明确承认的认知偏差。

## 二、批评

### 2.1 反对者的框架完全忽视了跨学科视角提供的机制解释

这是跨学科者最直接的批评：反对者在挑战所有三个假设时，仅使用了 ML 领域内的证据（TTT++、Sys2Bench 等），完全没有考虑到来自其他领域的结构性解释。这导致了一个重要的盲区：

**盲区 1（Assumption 1 的挑战）**: 反对者引用"DLMs 收敛到自回归式解码"（Li et al., 2026）来支持"跨步信息岛问题并不严重"的论点。但跨学科者的 turbo 编码框架给出了更精确的诊断：**关键不是跨步信息是否存在，而是跨步信息是否是外生的（extrinsic）而非自生的（intrinsic）**。如果 TTT 层只是重新编码骨干网络已经知道的信息，那么无论信息岛问题是否严重，TTT 层都没有贡献。这是一个比"信息岛是否存在"更细致的问题——而反对者的框架无法捕捉这个区别。

**盲区 2（Assumption 2 的挑战）**: 反对者正确地批判了"SSL 损失是任务性能的有效代理"这个假设，但没有提供为什么这个代理会失败的机制解释。跨学科者的免疫学类比提供了一个可测试的机制："原始抗原罪"（original antigenic sin）——早期低质量的 TTT 更新（高掩码率时）产生有害的归纳偏差，在后期低掩码率时持续干扰。这个机制预测：**如果从 r_c = 0.6 开始（丢弃早期更新）重新初始化快速权重，SSL-任务对齐将改善**。这是一个反对者的批判框架无法生成的可测试预测。

**盲区 3（Assumption 3 的挑战）**: 反对者引用统计物理证据来支持"更多计算不可靠地改善 DLM 推理"，但没有推进到下一步：如果这个限制存在，它的物理原因是什么，以及如何设计绕过它的方案？跨学科者的"临界减速"（critical slowing down）框架提供了这一步——临界区域是计算最有价值的地方，而在临界区域外的计算（无论是 ReMDM 的过度重掩码还是 DaL 的全步骤 TTT）都是低效的。

### 2.2 对"训练无关方法优于 DaL"的论证缺乏 baseline_remdm 结果的整合

反对者积极推荐训练无关方法（A-CFG +12.5pp，n=16），但新的 pilot 数据显示了一个复杂的图景：
- A-CFG（之前迭代）: +12.5pp（n=16）
- ReMDM: -9pp（baseline_remdm）
- info-gain soft: +5pp（alt_a_pilot，更可靠，n 更大）

这三个数据点共同表明：**训练无关方法的效果高度方法特异性**——并非所有训练无关方法都有益，ReMDM 的负向结果表明某些方法甚至有害。反对者当前的论证（"训练无关 > DaL"）在 ReMDM -9pp 的反例面前需要修正为"**正确设计的**训练无关方法 > DaL"，并且需要给出"什么是正确设计"的理论判据。

跨学科者认为，turbo 编码的外生信息原则提供了这个判据：训练无关方法应该提供**对骨干网络的外生补充信息**（info-gain soft 通过改变解码顺序来做到这一点——它让骨干网络在拥有更多上下文时再决定关键 token，这是外生的），而不是**内生地重新处理现有信息**（ReMDM 通过重掩码来做到这一点——它让骨干网络重新处理已经处理过的信息，这是内生的，符合 turbo 编码的"内生反馈导致 turbo cliff"预测）。

这个理论区分不仅解释了为什么 ReMDM 有害，也预测了哪些其他训练无关方法会有益（外生信息来源）或有害（内生重处理）。

### 2.3 "负面结果论文"策略没有充分考虑跨学科机制解释的发表价值

反对者建议将 22+ 次迭代的负面结果打包成一篇"为什么 TTT 在 DLMs 上失败"的论文。跨学科者认为，这个策略的发表价值高度依赖于是否提供了可普遍化的机制解释：

**低价值版本**: "我们尝试了 X、Y、Z，它们都不行，以下是失败日志。"这类论文在顶级会议上几乎不会被接受。

**高价值版本**: "通过跨领域分析（turbo 编码、统计物理、免疫学），我们发现 DLM 推理时计算扩展存在三个结构性障碍：(1) 内生信息反馈导致性能退化（turbo cliff，解释 ReMDM -9pp）；(2) 临界区域外的计算浪费（over-annealing，解释为什么均匀分配计算预算不如信息增益集中分配）；(3) 早期低质量更新产生有害归纳偏差（antigenic sin，解释 P3 失败）。我们提供了针对这三个障碍的诊断工具，并在 alt_a pilot 中展示了绕过障碍（1）和（2）的成功案例。"

后一种版本将负面结果转化为正向的机制理解，发表价值极高。反对者的框架只能生成前一种版本，需要跨学科者的机制解释来升级到后一种版本。

### 2.4 预测 D0c r < 0.15 的具体数字缺乏统计依据

反对者明确预测"D0c Pearson r < 0.15（远低于 0.3 阈值）"。跨学科者注意到，这个具体数字是从哪里来的？反对者的论证支持的是"r 可能很低"，但具体到 0.15 这个数字没有依据。

这个问题不是学术鸡毛蒜皮——如果 D0c 结果显示 r = 0.25（介于 0.15 和 0.30 之间），反对者的预测是"wrong"，但实验结论（SSL-任务对齐仍然不足）可能是"right"。预测过于具体（0.15 而非"< 0.30"）增加了被反证的风险而没有增加科学价值。好的科学预测应该精确到足以可证伪但不应该比证据支持的更精确。

## 三、评分

**整体评分**: 7.5/10

**分项评分**:
- 批判论证质量: 8.5/10（三个挑战假设证据扎实）
- 跨学科开放性: 3/10（完全忽视了跨学科机制解释）
- 新 pilot 证据整合: 7/10（ReMDM -9pp 验证了 C3，但与跨学科机制解释的联系未被识别）

## 四、核心建议

**建议 1（最高优先级）**: 将 ReMDM -9pp 结果纳入跨学科机制框架（turbo 编码的"内生反馈"导致 turbo cliff）来解释为什么 ReMDM 有害。这个解释可以预测哪些其他训练无关方法会有类似问题，具有超越本项目的普遍化价值。

**建议 2（高优先级）**: 将"负面结果论文"策略升级：不是"我们尝试了 TTT 它不行"，而是"我们发现了 DLM 推理时计算扩展的三个结构性障碍，并提供了基于跨学科分析的诊断工具"。这需要反对者与跨学科者的观点整合，但发表价值从"可能接受"升级为"强烈推荐"。

**建议 3（中优先级）**: 修正 D0c 预测为区间估计（例如，"预测 r 落在 [0, 0.25] 区间，而非 [0.25, 0.5]"）而非点估计（r < 0.15）。这同样是可证伪的，但允许更大范围的可能性。

**建议 4（低优先级）**: 尝试用"外生 vs 内生信息"框架重新表述 C2 假设：不是"SSL 损失是无效代理"，而是"SSL 损失优化的是内生信息重编码而非外生信息补充，因此根本上无法改善任务性能"。这个重表述是更精确的，且可以通过测量 TTT 输出与骨干输出的余弦相似度（跨学科者的 H_turbo 假设）来直接验证。

反对者与跨学科者的视角在深层次上是高度互补的：反对者提供了"什么不行"的实证证据，跨学科者提供了"为什么不行"的机制解释。两者的综合比任何单一视角都更有力，也更有发表价值。
