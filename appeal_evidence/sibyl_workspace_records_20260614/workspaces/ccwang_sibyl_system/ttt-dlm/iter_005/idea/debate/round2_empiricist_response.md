## 一、对跨学科者批评的回应

### 1.1 对批评 1 的回应：「D0c 无法区分三种失败原因（A/B/C）」

跨学科者提出最重要的批评：D0c 在当前设计下，失败时只能告诉我们「有问题」，无法区分三类根本不同的失败原因：
- **原因 A**（Turbo 内禀反馈）：TTT 层学习内生信息，SSL 改进与任务无关
- **原因 B**（Antigenic sin）：早期低质量伪标签污染 TTT 权重
- **原因 C**（结构性预测编码失配）：监督信号梯度方向与任务正交

建议在 D0c 中添加三个并行诊断测量（+3 GPU 小时），以区分原因 A/B/C。

**实验主义者的回应：部分接受，但有执行层面的修正**

跨学科者的核心批评是正确的：一个设计良好的诊断实验应该在「是否失败」之外也告诉我们「如何失败」。这从信息论的角度说，是最大化实验的「信息增益」——在相同计算预算下，区分多个假设比仅仅判断一个假设更有价值。

**然而，实验主义者有三个具体保留意见**：

**保留意见 1：+3 GPU 小时的成本估计是过于乐观的**。跨学科者建议的三个诊断分别是：余弦相似度测量（诊断 A）、检查点重启消融（诊断 B）、mini-VDS-TTT（诊断 C，替换 SSL 损失为答案验证器）。余弦相似度测量（+1 GPU 小时）是准确的。但 mini-VDS-TTT 不是在 D0c 配置 C 中「添加」一个测量，而是实现一个全新的训练目标（验证器加权的一致性损失）。这需要代码实现、调试、以及与原始 D0c 的对比——现实的成本是 +3-4 GPU 小时，不是 +1 GPU 小时。将 D0c 最小版本从 8 GPU 小时扩展到 11-12 GPU 小时，几乎是 50% 的成本增加。

**保留意见 2：三种失败原因不是完全独立的**。原因 A（内禀反馈）和原因 B（Antigenic sin）可能同时存在，且相互放大。在同一个 D0c 配置 B 上同时做「检查点重启消融」（诊断 B）和「余弦相似度测量」（诊断 A），可以揭示两者的相互作用，但解释起来比「原因 A 或原因 B」更复杂。实验主义者的偏好是：先做最便宜的诊断（余弦相似度测量 +1 GPU 小时），根据结果决定是否进行更昂贵的诊断（mini-VDS-TTT +3-4 GPU 小时）。这是序贯贝叶斯更新，不是同时并行。

**保留意见 3：原因 C（结构性梯度失配）的 mini-VDS-TTT 测试改变了 DaL 的根本假设**。如果配置 C 中替换 SSL 损失为验证器信号后准确率改善，这证明的是「VDS-TTT 风格的方法有效」，而不是「修复后的 DaL 有效」。这是一个值得独立研究的问题（VDS-TTT in DLMs），不适合作为 D0c 的诊断附属。将其纳入 D0c 会混淆「诊断 DaL 失败原因」和「测试新方法」这两个不同的实验目标。

**实际更新**：实验主义者接受在 D0c 中添加余弦相似度测量（跨学科者的 E_turbo 基础版），作为诊断原因 A 的轻量级扩展（+1 GPU 小时）。同时，在 D0c 的一个配置中添加检查点重启消融（+1 GPU 小时），作为原因 B 的测试。mini-VDS-TTT 作为独立的后续实验（如果 D0c 确认原因 C 最可能），不纳入 D0c 本身。总计 D0c 扩展成本：+2 GPU 小时，而非 +3 GPU 小时。

### 1.2 对批评 2 的回应：「Proposal 4 忽略了 Greedy 失败教训，缺少 tau-rho 联合搜索」

跨学科者指出 alt_a_pilot 已经明确显示：Info-Gain Greedy（-18pp）灾难性失败，而 Info-Gain Soft（tau=1.0，+5pp）成功。这揭示了随机性（退火率）对解码顺序的关键作用。在 A-CFG + Info-Gain Soft 的配对中，rho 和 tau 的联合效应需要系统搜索。

**实验主义者完全接受这个批评**，且这是一个真正的设计缺陷。

原始 Proposal 4 的 Phase 2（配对组合评估）确实没有将 (tau, rho) 联合网格搜索纳入计划。从统计学角度，这相当于忽略了一个重要的交互效应（interaction term）。如果 A-CFG 和 Info-Gain Soft 的效果是**相加的**，则联合搜索不是必要的。但如果它们存在**乘性交互**（例如，高 rho 的 A-CFG 与高 tau 的 Info-Gain Soft 相互抵消随机性），则独立调校后组合可能表现更差。

**具体更新**：Proposal 4 的 Phase 2 修改为：在 {(tau, rho)} = {(0.5, 0.5), (1.0, 0.5), (1.0, 1.0), (2.0, 1.0)} 的四点联合搜索上评估 A-CFG + Info-Gain Soft 的组合。这四个点覆盖了「两者都保守」、「Info-Gain 主导」、「均衡」、「Info-Gain 宽松」四种模式，足以检测主要的交互效应，计算成本约 +4 GPU 小时（4 个配置 × GSM8K-100）。

从跨学科者的模拟退火框架来看，这是在寻找「最优整体退火曲线」，而不仅仅是「两个独立参数的最优值」。实验主义者采用这个框架作为 Phase 2 设计的理论动机——它预测存在一个 (tau, rho) 的最优联合区域，而不是可分离的最优点。

### 1.3 对批评 3 的回应：「统计检验力分析结论未导出正确的实验规模」

跨学科者指出：实验主义者自己的检验力分析（n=200 对 +3-5pp 效应只有 35% 检验力）与 Proposal 4 使用 n=200 之间存在矛盾。

**实验主义者完全接受这个批评**。这是一个内部一致性问题，是实验设计中的真实缺陷。

**修订**：Proposal 4 的 Alt A 个体基线评估（Day 1）修改为使用 GSM8K-300 而非 GSM8K-200。理由：
- n=300 在基准准确率 40%、alpha=0.05 下，对 +5pp 效应有约 55% 检验力（仍不够高，但比 n=200 的 35% 有实质改善）
- 到达 80% 检验力需要 n=500（实验主义者原始提案中已提及，但没有落实到具体设计）

实验主义者承认：在当前 GPU 资源约束下（4 GPUs），使用 n=500 的成本约 40 GPU 小时，是 n=200（16 GPU 小时）的 2.5 倍。决策框架如下：
- **如果 Alt A 是主要研究轨道（D0c 失败后）**：使用 n=500 以确保统计有效性
- **如果 Alt A 是并行补充轨道（D0c 进行中）**：使用 n=300 作为初步信号，明确标注「统计确认需要 n=500」

这个区分是明确的。原始提案没有做出这个区分，是一个疏漏。

### 1.4 对批评 4 的回应：「负面结果的报告需要机理解释层」

跨学科者指出「TTT-MLP 在 GSM8K 上不如 vanilla」这样的负面结果如果没有机理解释，在顶级会议上不够。需要理论家的 Delta_I 测量和跨学科者的 Turbo 余弦相似度测量。

**实验主义者接受这个批评的核心**，但需要澄清一个重要的分工问题：

实验主义者的价值在于提供**严格的测量和统计设计**，而机理解释层是**理论家和跨学科者**的比较优势。实验主义者已经提出了「update rule ablation publishable」的概率为 60%，这个估计的前提是「有适当的机理解释配合」。如果没有机理解释，这个概率更接近 30%。

**具体更新**：实验主义者在负面结果论文计划中，明确要求以下三个机理测量作为预先指定（pre-specified）的附加实验：
1. 余弦相似度诊断（跨学科者的 E_turbo，+2 GPU 小时）
2. Delta_I 估计（理论家的 D0d，+8 GPU 小时）
3. 跨 backbone 验证（Dream-7B + LLaDA-8B，+8 GPU 小时）

这三个实验的目的不是「修复 DaL」，而是为负面结果论文提供机理解释层，将其从「我们试了，不管用」升级为「我们发现了结构性障碍」。这是合理的实验规划调整，成本约 +18 GPU 小时，但发表价值的提升是值得的。

## 二、对创新者批评的回应

### 2.1 对批评 2.1 的回应：「D0c 框架没有探索替换 SSL 目标的路径」

创新者指出 D0c 的决策树在「r < 0.3 → 杀死 DaL」时，没有考虑「替换 SSL 目标」（如用信息增益损失替换 MLM 损失）的分支。D0c 只能判断「当前 SSL 目标是否有效」，不能判断「所有可能的 SSL 目标是否都无效」。创新者建议：alt_a_pilot 的 +5pp 暗示信息增益信号可能是更好的 TTT 目标。

**实验主义者部分接受，但有重要的方法论保留意见**：

创新者的基本观点是正确的：D0c 测试的是「当前 MLM 对齐假设是否成立」，而不是「所有 TTT 方法是否都无效」。如果 r(SSL, acc) < 0.3，正确的推断是「当前 MLM SSL 目标不对齐」，而不是「所有 TTT 框架都不行」。

然而，实验主义者对「将信息增益损失作为 TTT 目标」的提议持谨慎态度，原因如下：

**方法论问题 1**：信息增益解码（alt_a_pilot 的成功）是一种**推理时策略**（改变解码顺序），而不是**训练目标**（训练 TTT 权重）。将「解码时信息增益有效」外推到「训练时信息增益损失也有效」是一个非平凡的跳跃。这两者在技术上是完全不同的：
- Info-Gain Soft 在推理时不更新任何权重，只改变下一个解掩蔽的 token 选择
- 「信息增益 TTT」需要在训练时计算信息增益梯度，这在技术上并不直接（信息增益需要对整个分布积分，梯度计算不trivial）

**方法论问题 2**：创新者的建议实质上是提出一个新方法（信息增益 TTT），而不是修复现有方法（DaL）。这是两个不同的研究项目。实验主义者的框架是系统地测试和排除，而不是无限地扩展测试空间。如果每次 D0c 失败都产生一个「让我们测试另一种 SSL 目标」的提议，项目将永无止境。

**有限度的更新**：实验主义者在 D0c 决策矩阵的「r < 0.3，r(gate, acc) > 0.3」单元格（SSL 损失不对齐但门值有效）添加一个**有限的**备选路径：「测试 2 个替代 SSL 目标（VDS-TTT 风格验证器信号，以及自洽性损失），评估 GSM8K-50，不超过 4 GPU 小时。如果任一替代目标在 GSM8K-50 上 > vanilla + 3pp，则继续探索；否则，全面转向 Alternative A」。

这个备选路径是严格有界的（2 个候选目标，4 GPU 小时，单一扩展决策）。它不是「无限扩展 SSL 目标空间」，而是「在清晰的停止条件下测试最有理论支持的备选方案」。

### 2.2 对批评 2.2 的回应：「对训练无关方法成功可能性的系统性低估」

创新者认为 Alternative A 的成功概率应该从 45% 上调到至少 65%，理由是 Gini=0.851 PASS 排除了退化解释，+5pp 改进有统计支持。

**实验主义者维持保守估计，但承认需要更新先验**：

Gini=0.851 确实排除了 degenerate collapse（这是低估风险中的一个具体可能性）。但以下不确定性仍然存在：

1. **n=100（GSM8K-100）的统计力不足**：按照实验主义者的检验力分析，n=100 在 +5pp 效应下的检验力约为 50%（即使真实效应存在，也有 50% 概率在 n=100 时不显著）。「有正向信号」不等于「统计确认」。
2. **组合时的不稳定性**：info-gain soft 单独有效（+5pp），但与其他方法组合时的效果未知。ReMDM 是一个警示——一个看起来合理的方法实际上有害。
3. **泛化性未知**：GSM8K 的 +5pp 是否在 ARC-C、HumanEval 上也成立？alt_a_pilot 只测试了 GSM8K-100。

**折中更新**：实验主义者将 Alternative A 的成功概率从 45% 上调至 55%（而非创新者建议的 65%），主要根据 Gini 检验排除了退化解释。剩余的不确定性来自统计检验力不足和泛化性未知。

实验主义者保留了「A-CFG +12.5pp（n=16）是噪声」的判断——n=16 的 95% CI 约为 ±15pp，这意味着真实效应可能在 [-2.5pp, +27.5pp] 范围内。这个不确定性不能被简单地「升级」成强正向信号。

### 2.3 对批评 2.3 的回应：「没有将 alt_a_pilot 转化为新研究假设」

创新者建议将 info-gain soft 的 +5pp 升级为第一优先研究对象，专门设计「信息增益解码机制分析」实验来回答「为什么有效」。

**实验主义者原则上同意，但在优先级排序上有保留**：

「为什么信息增益解码有效」是一个重要的科学问题，有高价值的机理解释意义。但实验主义者的核心哲学是：先测量效应的稳定性和规模（以 n=200+ 确认 +5pp），再投入资源研究机理（「为什么」）。

**如果我们先做「为什么分析」而 n=200 验证后发现 +5pp 不可重复（统计噪声），则机理分析是浪费**。实验主义者的序贯设计原则：效应确认 → 机理理解，不是 → 机理理解 → 效应确认。

**实际更新**：实验主义者将以下修改加入计划：
- Alt A 的 Day 1 评估扩展到 n=300（而非 n=200，如批评 3 所述）
- 如果 n=300 结果显示效应稳定（+4pp 以上），则在 Day 2/3 添加机理分析实验（信息增益为何有效：解码顺序的困惑度分析、token 先后揭示的条件互信息测量），约 +4 GPU 小时
- 机理分析的触发条件是效应确认，而不是提前进行

这个安排是「确认后分析」模式，与实验主义者的核心哲学一致。

### 2.4 对批评 2.4 的回应：「没有对 alt_a 验证实验进行正式的样本量规划」

创新者指出确认 +5pp 改进所需的 n（约 n=150 可以 80% 检验力）应该被明确纳入 GPU 计划。

**实验主义者完全接受这个批评**，且这与跨学科者批评 3 的更新一致。

**具体更新**：
- 正式计算：基准准确率 40%，检测 +5pp 改进（Delta=5pp），alpha=0.05，power=80% 所需 n≈135
- Proposal 4 的 Alt A 基线评估使用 n=300（覆盖 n=135 所需，同时允许多任务泛化性检验）
- 明确标注：如果 n=300 结果的效应量 Cohen's h < 0.1（非常小的效应），则即使 p < 0.05，也不应在论文中报告为「统计显著改进」

## 三、立场更新

基于跨学科者和创新者的批评，实验主义者对原始提案做出以下更新：

**维持的立场**：
- D0c 预注册决策矩阵的设计不变（GO/NO-GO 二元决策，无「方向性正向」例外）
- 配对 bootstrap + Holm-Bonferroni + Cohen's d 的统计规范不变
- Alternative A 与 DaL 诊断**并行**运行的策略不变（不等 D0c 结果再启动 Alt A）
- 「效应确认优先于机理分析」的序贯原则不变

**修订的立场**：
- **D0c 扩展**：添加余弦相似度测量（+1 GPU 小时）和检查点重启消融（+1 GPU 小时）作为原因 A/B 诊断。mini-VDS-TTT 作为独立后续实验，不纳入 D0c 本身
- **D0c 决策矩阵扩展**：在「r < 0.3，r(gate) > 0.3」单元格添加有限备选路径（测试 2 个替代 SSL 目标，≤4 GPU 小时，有明确停止条件）
- **Alt A 样本量**：从 n=200 改为 n=300（信号筛选），以 n=500 作为「统计确认」标准
- **Proposal 4 Phase 2**：添加 (tau, rho) 联合网格搜索（4 个配置，+4 GPU 小时）
- **成功概率更新**：Alternative A 成功概率从 45% 上调至 55%（基于 Gini 检验排除退化）
- **负面结果论文**：预先指定 3 个机理测量实验（E_turbo 余弦相似度、D0d Delta_I、跨 backbone 验证）作为提高发表质量的必要附加实验

**新增的计划元素**：
- 机理分析触发条件：Alt A n=300 确认 +4pp 以上效应后，启动「信息增益解码机制分析」（解码顺序的条件互信息测量，+4 GPU 小时）

## 四、共识点

经过跨学科者和创新者的批评，以下几点达成实质共识：

1. **D0c 的信息提取可以增强**：通过添加余弦相似度测量和检查点重启消融（+2 GPU 小时），在相近成本下获得对失败原因的诊断信息

2. **Alt A 需要更大样本量**：n=200 对 +5pp 效应只有 35% 统计检验力，实验主义者的检验力分析需要落实到实验规模（n=300 信号筛选，n=500 统计确认）

3. **tau-rho 联合效应是真实的设计风险**：Greedy（-18pp）vs Soft（+5pp）的对比证明随机性参数对解码结果影响巨大，Proposal 4 Phase 2 需要联合搜索

4. **信息增益成功背后有机理解释价值**：alt_a_pilot 的 +5pp 不仅是工程发现，且有跨学科框架支持的机理意义，值得在效应确认后专门研究

5. **负面结果论文需要机理层**：三方均认为仅有失败记录不够，需要 Delta_I（理论家）、Turbo 诊断（跨学科者）、跨 backbone 验证共同支撑
