**被评审者**: sibyl-innovator（创新者）
**评审轮次**: Round 1, Batch 2, REFINE Round 3

**新增 pilot 证据**:

- alt_a_pilot: info-gain soft 39%（+5pp），Gini=0.851 PASS

## 一、优点

### 1.1 对 DaL 失败根因的分析切中要害，多个创新提案有其合理性

创新者正确地识别了 P3 失败的三个层面：(a) gate 卡住（工程问题），(b) SSL 信号不到达骨干网络（传播问题），(c) 即使到达也不知道是否有帮助任务（对齐问题）。这三层分析比仅关注"修复 gate"的工程视角更为全面。特别是提案 1（ADM：关联记忆）的数学动机令人印象深刻——Ambrogioni (2024) 将扩散模型与现代 Hopfield 网络联系起来的工作如果成立，那么"扩散去噪 = 模式检索"这个框架确实为不需要梯度的推理时扩展提供了新的方向。

### 1.2 明确列出失败模式和成功概率是科学诚信的表现

创新者在每个提案下都明确列出了"失败模式"（Failure Modes）和"成功概率"（30%、35%、20%、25%、25%）。相比其他视角倾向于过度乐观，这种概率化的不确定性表达更接近真实的科学态度。特别是对 LDR（潜在扩散推理，Proposal 3）的"20% 成功概率"估计，结合其"16-20 GPU-hours"的成本，给出了期望价值分析的基础。

### 1.3 新 pilot 证据对创新者提案组合的影响被适当识别

创新者提到的"Best-of-N 完全无效"历史（3x 计算，+6.9%，可能是噪声）在新 pilot 证据下得到了额外支持：ReMDM -9pp 表明简单的重采样类方法不仅无效，甚至有害。这强化了创新者的核心直觉——需要真正新颖的方法，而不是现有方法的组合。

## 二、批评

### 2.1 所有五个提案都缺少预注册的杀死标准（kill criteria）——这是实验设计的致命弱点

实验主义者的最大不满：创新者提供了"失败模式"的定性描述，但没有**预注册的定量 kill criteria**。这个缺失在新 pilot 证据的背景下尤为严重：

**例子 1（ADM）**: "Hopfield 检索可能产生模糊/平均化的模式（mode collapse）"是一个定性描述，但什么是可测量的 kill criterion？如果 GSM8K 准确率在 2 GPU-hours 后没有超过 vanilla + 1%，是否立即停止？如果输出的 Distinct-1/2 低于某个阈值（说明 mode collapse），是否停止？

新的 pilot_entropy 结果（Gini=0.851 PASS）展示了正确的实验设计——有明确的分布健康性指标。创新者的每个提案都应该有类似的预注册质量指标。没有这个设计，ADM pilot 即使产生了负面结果，也可能被"后验合理化"为"需要更多训练步骤"或"超参数没有调好"，导致资源浪费。

**例子 2（SPD：自博弈去噪）**: "DPO 梯度信号可能太嘈杂"是失败模式描述，但没有说明在什么条件下停止——如果 4 次轨迹中 winning trajectory 和 losing trajectory 的分数差异小于某个阈值，DPO 更新本质上是在噪声上做梯度下降，应该立即 abort。

### 2.2 ADM（提案 1）的"数学动机"存在从理论到实践的巨大跳跃

实验主义者专门注意到创新者引用的 Ambrogioni (2024) 关于"扩散模型 = 关联记忆网络"的等价性。这个理论结果是**渐近等价**（asymptotic equivalence），仅在无限参数、无限数据的极限下成立。对于有限规模的实际 Dream-7B 模型，这个等价性是否成立完全未知。

更重要的是，实验主义者注意到：**即使这个等价性成立，它也是描述性的（diffusion IS Hopfield），而不是处方性的（adding Hopfield layers WILL improve diffusion）**。一个系统等价于 Hopfield 网络并不意味着向其中插入显式 Hopfield 层会改进它——这类似于说"神经网络是统计学习器，所以向神经网络中添加统计量提取器会改进它"，这个逻辑显然不成立。

**具体可测试的前置条件**: 在运行任何 ADM GPU 实验之前，需要先测量 Dream-7B 骨干 hidden states 的自相关结构（第 T 步的 hidden states 和第 T-k 步的 hidden states 之间的相关性）。如果相关性低于 0.1，那么 Hopfield "检索"从早期步骤的 hidden states 中能提取的有效信息接近于零，整个 ADM 方案在出发点上就是错误的。这个测量成本 < 1 GPU-hour。

### 2.3 对 alt_a_pilot 正向信号的利用不足，而新奇性偏好导致资源分配失当

创新者的框架隐含了一个新颖性偏好（novelty bias）：ADM (30%)、SPD (25%)、PC-DLM (25%) 的成功概率都明显低于实用主义者方案（COBA 65%、EGF 55%），但创新者仍然优先推荐这些高风险方案，理由是"更高新颖性 = 更高影响力"。

**新 pilot 证据对这个偏好的质疑**: alt_a_pilot 的 info-gain soft +5pp 是一个**中等新颖性但高确信度**的发现（Gini=0.851 PASS 表明不是 degenerate，+5pp 在 pilot 规模上已经统计上可信）。在这个情况下，"低新颖性但高可靠性"的方向（扩展 alt_a 到 n=300 验证和机制消融）的期望价值可能高于"高新颖性但低可靠性"的方向（ADM pilot）。

**期望价值估算**:
- alt_a 扩展（2 GPU-hours，估计确认成功概率 70%，+5pp 改进）: E[改进] = 0.70 × 5pp = 3.5pp/2h = 1.75pp/GPU-hour
- ADM pilot（4 GPU-hours，估计成功概率 30%，+2pp 改进）: E[改进] = 0.30 × 2pp = 0.6pp/4h = 0.15pp/GPU-hour

即使采用创新者自己的数字，alt_a 扩展的期望价值也约是 ADM pilot 的 12 倍。在 4 GPU 有限的资源下，**应该优先扩展已有正向信号，而不是探索新颖但低概率的方向**。

### 2.4 SPD（自博弈去噪，提案 4）对 DLM 并行性的假设可能被 pilot 证据否定

创新者声称"SPD 利用 DLM 的天然并行性：多条轨迹可以并行运行"，并将此作为 SPD 相比 AR 模型自博弈的优势。但实验主义者注意到一个被忽略的复杂性：

**问题**: DLM 的"并行去噪"产生的多条轨迹是否足够多样（diverse）以提供有效的对比学习信号？或者，由于 DLM 的确定性去噪倾向（Dream-7B 使用 Top-p 采样），多条轨迹可能高度相似，使得 SPD 的 "winning vs. losing trajectory" 对比退化为噪声？

**新 pilot 证据的间接警告**: ReMDM -9pp 表明重采样（remasking）策略在某些情况下会产生比 vanilla 更差的输出。如果 ReMDM 的重采样产生了"差的轨迹"（选择了低质量的重掩码模式），那么 SPD 的"自我奖励"（用 DLM 自身的 log-likelihood 作为奖励）也可能选择了"低质量但高置信度"的轨迹，导致 DPO 更新朝向错误方向。这是创新者的"自奖励可能标定不准"失败模式的一个具体实例，需要在 pilot 设计中明确控制。

## 三、评分

**整体评分**: 6.5/10

**分项评分**:
- 创意和新颖性: 9/10（ADM、LDR、PC-DLM 都是真正新颖的方向）
- 实验设计严格性: 4/10（缺乏预注册 kill criteria，成功条件定义模糊）

- 资源期望价值: 4/10（高新颖性偏好导致资源配置低于最优）

## 四、核心建议

**建议 1（最高优先级）**: 为每个提案添加预注册的定量 kill criteria，格式如下："如果在 [X] GPU-hours 之内，[具体指标] 未达到 [具体阈值]，立即停止该方向"。这不限制创新，但确保创新不会成为无底洞。

**建议 2（高优先级）**: 在运行 ADM pilot 之前，先用 < 1 GPU-hour 的诊断实验测量 Dream-7B hidden states 的跨步相关性结构。如果相关性 < 0.1，ADM 的 Hopfield 检索无法提取有用信息，整个方向应该 abort。

**建议 3（中优先级）**: 将资源分配重新平衡，将 alt_a 扩展验证（n=300）列为第一优先级（而不是"继续 Alternative A"的边注）。现有正向信号应该优先得到验证，高新颖低概率方向应该在资源允许时追求，不应该抢占已有正向信号的验证资源。

**建议 4（低优先级）**: SPD 方案在实施前应该先测量 Dream-7B 在相同输入下多条轨迹的 token-level diversity（Distinct-1/2）。如果多条轨迹高度相似（Distinct-1 > 0.9），SPD 的对比学习信号将是噪声占主导，整个方案的前提假设不成立。

创新者的视角是项目中最具创造力的，但创造力必须与实验设计的严格性相结合才能产生科学价值。在新 pilot 证据的约束下，最有价值的创新是对已有正向信号（info-gain soft）的机制理解，而不是探索全新的 ADM 或 PC-DLM 方向。
