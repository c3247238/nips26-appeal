# 创新者评实用主义者

## 总分: 5.8/10

## 各维度评分

- 创新性: 4/10 - AdaReMask 的核心想法——用熵/置信度/翻转频率来选择重遮蔽位置——本质上是对 ReMDM-conf 和 TCR 的线性组合。将三个已知信号做加权求和（U_i = alpha*H + beta*(1-c) + gamma*F）不构成根本性新想法，更像是一个超参数搜索问题。与 Prism 的自验证、PG-DLM 的粒子 Gibbs、甚至 RemeDi 的双流架构相比，AdaReMask 缺乏一个让人眼前一亮的新机制或新视角。"token 级自适应"这个卖点，DiFFPO 已经初步做了，差异仅在粒度上，不够颠覆。

- 技术深度: 5/10 - 方法的技术贡献主要集中在工程实现层面（怎么记录熵轨迹、怎么分桶、怎么提前终止），而非算法或理论层面。综合不确定性评分 U_i 的三个分量没有理论动机说明为什么这三个信号足够且最优。为什么是线性组合而非其他融合方式？alpha, beta, gamma 的最优值是否有闭式解？分层去噪步数的分桶策略为什么比连续分配更好？这些问题都没有回答。与信息瓶颈框架或能量模型视角相比，技术深度明显不足。

- 可行性: 8/10 - 这是该提案最大的优势。实验路径极为清晰，从 Phase 0 到 Phase 4 层层递进，时间估算合理（20-29 天），计算需求可控（~100 GPU 小时），代码复用度高（TCR v1 框架可直接扩展）。风险分析全面且务实，每个风险都有对应的缓解策略和备选方案。特别是"最坏情况备选方案"（转向系统性基准测试论文）体现了成熟的项目管理思维。

- 顶会潜力: 5/10 - 在当前形式下，投 NeurIPS/ICML/ICLR 主会困难较大。核心问题是缺乏足够的 intellectual novelty——reviewer 很可能评价为"straightforward combination of existing ideas"。Training-free 是加分项，但 ReMDM 也是 training-free 的。如果实验结果非常强（在多个 benchmark 上显著超越所有基线），有可能勉强过关，但提案自身对效果不够的风险评估概率高达 40%（风险 1）和 30%（风险 2），说明作者自己也不太确信能有足够强的数值结果。目标可能需要降级到 EMNLP/ACL 或 workshop。

- 风险评估: 7/10 - 风险分析部分是整个提案中做得最好的。四个风险的识别准确（training-free 无效、benchmark 无提升、计算过大、被抢发），概率估计合理，缓解策略具体可操作。但缺少一个关键风险的讨论：综合评分 U_i 中三个分量之间可能存在高度共线性（高熵的 token 很可能也有低置信度和高翻转率），导致组合带来的信息增益极其有限，本质上等价于只用其中一个信号。

## 主要优点

1. **极高的工程可行性**：从已有代码框架出发，增量式实现，风险最低。TCR v1 的推理管线和 logits hook 机制直接复用，核心算法仅 300-500 行 Python，这是一个两周内就能拿到初步结果的方案。

2. **务实的评测策略**：吸取了 18 轮迭代中"PPL 不可信"的教训，从设计之初就以 benchmark 准确率为唯一评判标准。评测方案覆盖数学推理、代码生成、规划等多个维度，有消融实验、效率分析和可视化，实验设计本身是高质量的。

3. **优雅的退化性质**：当所有 U_i < tau 时，AdaReMask 退化为 vanilla 推理。这保证了方法在最坏情况下不会比基线差，给予实验很大的安全边际。

4. **与 TTT 的联系有启发性**：将去噪轨迹视为"过程记忆"、将重遮蔽决策视为"哪些记忆需要更新"的类比，虽然不够形式化，但为后续工作提供了有价值的直觉。

## 主要不足

1. **缺乏根本性创新机制**：整个方案可以概括为"用多个已知信号的线性组合替代单一信号来选择重遮蔽位置"。没有引入新的计算范式（如能量引导、MCMC 接受/拒绝、粒子采样），没有新的理论视角（如信息瓶颈、变分推断），也没有跨领域的思想迁移。这在顶会 review 中几乎必然会被质疑 novelty。

2. **贪心接受的老问题未解决**：AdaReMask 仍然是贪心地接受重遮蔽后的结果——只要重遮蔽了就接受新的预测。这正是 ReMask-Retry 在 18 轮迭代中暴露的核心失败模式（PPL gaming、重复文本退化）的根源。提案中完全没有讨论如何防止重遮蔽后陷入新的局部最优。分层去噪步数分配和提前终止都无法解决这个根本问题，因为问题不在于"遮蔽多少"，而在于"接受什么样的重去噪结果"。

3. **U_i 三个分量的共线性问题**：直觉上，一个 token 如果熵高（H 大），那么置信度一定低（c 小）且翻转频率大概率高（F 大）。这三个信号高度相关，线性组合的信息增益可能极其有限。提案需要在 pilot 实验中验证三个分量的互信息，或者提供理论论证说明它们提供了互补信息。

4. **"分层去噪步数分配"缺乏理论支撑**：将重遮蔽 token 按不确定性分为 K 个桶并分配不同步数，这个策略缺乏理论依据。为什么分桶比连续分配好？桶的边界如何确定？步数与不确定性的最优映射关系是什么？这些都是 ad hoc 的设计选择，reviewer 会追问。

5. **与 Prism 的对比不够公平**：Prism 也是 training-free 的、也利用模型自身信号来引导搜索，且通过层次分支搜索实现了更精细的局部精炼。AdaReMask 相对于 Prism 的优势不明确——如果 Prism 在相同计算预算下效果更好，AdaReMask 的存在意义就大打折扣。

## 改进建议

1. **引入接受/拒绝机制**：借鉴 Metropolis-Hastings 准则，在重遮蔽-重去噪后不盲目接受新结果，而是比较新旧序列的质量（可用模型自身的伪似然评估），以概率接受或拒绝。这能从机制上解决贪心接受导致的局部最优问题，同时显著提升方法的理论深度。

2. **寻找更强的理论锚点**：当前的 U_i 公式是纯启发式的。建议从信息论出发推导：在什么条件下，熵+置信度+翻转频率的线性组合是最优重遮蔽准则的一阶近似？如果能证明 U_i 近似了某个信息论目标函数（如条件互信息），方法的说服力会大幅提升。

3. **验证分量独立性**：在 pilot 实验中计算 H、c、F 三者之间的 Pearson/Spearman 相关系数。如果高度共线（>0.9），则简化为单一最有效信号（大概率是熵 H），反而使方法更简洁有力。如果确有互补性，这本身就是一个有价值的实证发现。

4. **加入退火调度**：当前阈值 tau 是固定的。建议在多轮重遮蔽中引入递减的 tau（即退火），早期探索性大、后期精炼性强。这与模拟退火的理论保证相结合，能为方法提供渐近收敛的理论支撑。

5. **重新定位论文贡献**：如果实验证明 training-free AdaReMask 在多个 benchmark 上有一致的提升，建议将论文重新定位为"DLM 不确定性信号的系统性分析 + 自适应计算分配的实证研究"，而非"新方法"。这种定位更诚实，也更适合当前技术深度，可以投 EMNLP/ACL 主会。
