# 实用主义者对创新者方案（EGIR）的批判性评估

**评审对象**: EGIR（Energy-Guided Iterative Refinement）——将 DLM 去噪重构为能量景观搜索
**评审角色**: 实用主义者（工程可行性导向）

---

## 1. 创新性 — 7/10

**认可之处：**
- 将 DLM 去噪过程显式重构为能量景观搜索，这个视角转换是有价值的。它不是在现有 remasking 策略上做微调（如 ReMDM 的 cap/rescale/conf），而是提供了一个统一框架来理解和设计 remasking 策略。这种"元框架"思维比逐个策略试错高效得多。
- MH 接受准则的引入确实抓住了 ReMask-Retry 失败的核心机制——贪心接受导致模式坍塌。从工程角度看，这是对失败根因的精准修复，而非绕路。
- Surprise map 作为 remasking 信号，比单纯的置信度或熵更有信息量，因为它结合了模型对"当前 token 是否属于此处"的判断。

**保留意见：**
- "能量模型 + Langevin 动力学"的叙事包装略显厚重。剥去理论外衣后，EGIR 的核心操作是：（1）用伪似然打分，（2）按 surprise 选位置 remask，（3）退火调温，（4）概率接受/拒绝。这些组件本身并不新——伪似然在 BERT 时代已被广泛使用，退火和 MH 是经典 MCMC 工具。创新在于**组合方式和应用场景**，而非单个组件。
- 与 PG-DLM（粒子 Gibbs）的差异主要是"自评估 vs 外部奖励"，这个区别在实践中是否足够大，取决于伪似然能量函数的有效性——而这恰恰是最大的未知数。

## 2. 技术深度 — 7/10

**扎实之处：**
- 能量函数的设计考虑了两个层面：逐 token 伪似然（局部自洽性）和全局一致性项（prompt 对齐），这种分层设计是合理的。单独用 PPL/伪似然会遭遇 18 轮实验已暴露的"PPL gaming"问题，全局项的加入是必要的防护。
- 块伪似然的计算优化（B 块并行，B << N）是一个实用的工程决策，将 O(N) 次前向传播降到 O(B) 次。
- 三个变体（Lite/Full/Parallel）的设计体现了从 MVP 到完整方案的渐进思维，这对实际研发很重要。

**不足之处：**
- 全局一致性项 $H_{global}$ 的定义不够精确。"模型在给定完整生成 y 时对 prompt x 中关键信息 token 的注意力熵"——哪些是"关键信息 token"？如何确定？这涉及到注意力头的选择、层的选择等大量工程细节，提案中完全没有讨论。在实践中，这个组件的实现难度可能远超预期。
- MH 接受准则的理论保证（"温度趋零时收敛到能量最低状态"）在数学上是正确的，但在实际应用中几乎无意义：（a）我们不会让温度趋零（那样接受率也趋零）；（b）收敛速度未被分析——5 轮精炼（K=5）是否足够？混合时间是多少？
- 伪似然与真实似然的差距在 DLM 中有多大？Masked LM 的伪似然估计存在已知偏差（token 间条件独立假设），块伪似然是否能有效缓解？提案未讨论。

## 3. 可行性 — 6/10

**可行之处：**
- EGIR-Lite 的实现确实接近零门槛：利用已有 logits 作为 surprise 估计，加上退火 remasking 调度，代码量在 100-200 行。作为概念验证，这是合理的起点。
- 不需要训练、不修改架构，纯推理时的算法改进——这意味着可以直接在 Dream-7B 等开源模型上验证，无需额外资源。

**重大工程风险：**
- **计算开销被严重低估**。提案声称 EGIR-Full 计算量为标准去噪的 ~1.3x（B=4, K=5, T=64），但这个计算有误：
  - 逐块伪似然：每轮需要 B=4 次前向传播来计算能量
  - K=5 轮精炼，每轮包含：能量计算（4 次 FP）+ 局部重去噪（t_local 次 FP）+ 新状态能量计算（4 次 FP）
  - 保守估计：额外开销 = K × (2B + t_local) = 5 × (8 + 10) = 90 次前向传播
  - 标准去噪 T=64 次，总计 64 + 90 = 154 次，约 2.4x 而非 1.3x
  - 如果算上 MH 拒绝导致的无效计算，实际开销更高
- **全局一致性项的工程实现模糊**。注意力熵需要访问模型内部的注意力权重矩阵，这在不同框架（HuggingFace vs vLLM vs 自定义推理）中的实现方式差异很大。对于 7B 模型，提取和处理注意力矩阵的内存开销也不可忽视。
- **MH 接受率的调参噩梦**。退火调度有 3 个超参（$\tau_0$, $\tau_{min}$, 衰减形式），MH 接受还依赖能量函数的绝对尺度。如果能量函数的尺度在不同输入上差异很大（很可能如此），固定的温度调度将无法工作，需要自适应温度——又增加了一层复杂性。
- **Phase 1 的预期过于乐观**。"EGIR-Lite >= +2% accuracy on GSM8K"——考虑到 Dream-7B 在 GSM8K 上的基线准确率可能本就不高（文献中 DLM 在数学推理上普遍弱于同规模 AR 模型），2% 的绝对提升需要方法在正确方向上有显著效果。而 EGIR-Lite 去掉了两个核心组件（块伪似然和 MH），本质上退化为"退火版 ReMask"，与 ReMDM-conf 的差异有限。

## 4. 顶会潜力 — 7/10

**有利因素：**
- 框架的统一性和理论叙事是顶会喜欢的要素。"将 DLM 去噪重构为能量景观搜索"如果配合强实验结果，完全可以投 NeurIPS/ICML。
- 与多个热门方向交叉（能量模型、MCMC、扩散模型、测试时计算扩展），审稿人容易找到共鸣。
- 三个变体的设计让消融实验自然形成层次结构，论文结构友好。

**不利因素：**
- **实验结果是关键瓶颈**。理论框架再优美，如果 EGIR 在 GSM8K/MATH500 上的提升不显著（< 2-3%），审稿人会质疑"理论很好但实际没用"。18 轮实验的历史教训表明，DLM 上的推理时扩展方法极容易失效。
- **与 Prism 的竞争**。Prism 已在推理任务上展示了正面结果（层次搜索），且同样 training-free。如果 EGIR 不能在性能或效率上明确超越 Prism，差异化论点会被削弱。
- **"首次为 DLM remasking 提供渐近最优性保证"这一 claim 需要谨慎**。MH 的渐近收敛是在无限步数下的理论性质，实际 5-10 轮精炼不保证任何东西。审稿人可能会指出理论与实践的脱节。

## 5. 风险评估 — 6/10

**充分之处：**
- 识别了伪似然有效性、计算开销、MH 接受率三个高风险点，且给出了缓解策略。EGIR-Lite 作为最低可行方案的设计体现了风险意识。
- "DLM 基础能力不足"的中等风险也是务实的认知。

**严重遗漏：**
- **未识别最关键的风险：伪似然能量函数可能与下游任务质量无关**。18 轮实验最深刻的教训是"PPL 不可信"。伪似然本质上是 PPL 的变体（都是基于模型自身的似然估计），因此 EGIR 面临与 ReMask-Retry 完全相同的根本问题——模型自身的打分可能无法区分"真正好的生成"和"看起来好的生成"。提案用全局一致性项 $H_{global}$ 来缓解，但这个项的有效性同样未经验证。这是整个方案的阿喀琉斯之踵。
- **未讨论 MH 拒绝导致的"精炼停滞"问题**。如果大部分修改被拒绝（接受率 < 20%），K=5 轮精炼可能只有 1 轮有效更新，方法实质上退化为单步 remasking。提案提到"监控接受率并自适应调节温度"，但没有讨论如果自适应调温后接受率仍然低怎么办。
- **未评估与 AdaReMask（实用主义者方案）的重叠风险**。两个方案都使用模型自身的信号（EGIR 用伪似然/surprise，AdaReMask 用熵/翻转频率）来指导 remasking，核心思路有相当大的重叠。如果 AdaReMask 以更简单的方式达到类似效果，EGIR 的额外复杂性就不合理了。
- **实验时间线过于紧凑**。Phase 1（1-2 周）要完成概念验证，但仅超参搜索（$\lambda$, $\tau_0$, $\tau_{min}$, B, K, t_local）就有 6 个维度。在 GPU 资源有限的情况下，系统性搜索不可能在 1 周内完成。

## 总体判断

EGIR 是三个提案中**理论叙事最完整、框架最优美**的方案，能量景观搜索的视角确实为 DLM 的 remasking 问题提供了新的思考维度。但从工程可行性角度看，存在三个核心担忧：

1. **伪似然与 PPL 同源**：EGIR 的能量函数本质上是 PPL 的精炼版本，而 18 轮实验已经证明 PPL 不可信。全局一致性项 $H_{global}$ 是否能有效补偿这一根本缺陷，是整个方案成败的关键，但目前完全没有证据支撑。

2. **工程复杂度与收益不成比例**：完整版 EGIR-Full 涉及块伪似然计算、注意力熵提取、退火调度、MH 接受/拒绝等多个紧耦合组件，调参空间巨大。如果最终效果只比简单的"退火 remasking"（EGIR-Lite）好 1-2%，这些额外复杂性就不值得。

3. **计算开销被低估约 2 倍**：实际的 FP 次数分析表明 EGIR-Full 约为 2.4x 标准去噪，而非声称的 1.3x。这意味着 EGIR-Full 在效率上可能不比 Best-of-3 更优，削弱了其作为"高效推理时扩展"方法的定位。

**实践建议：**
- 优先实现 EGIR-Lite，但用 benchmark accuracy（而非 PPL/能量下降）作为唯一判断标准。如果 EGIR-Lite 的退火 remasking 在 GSM8K 上无提升，EGIR-Full 的额外组件大概率也无法挽救。
- 在投入 EGIR-Full 之前，先做一个快速实验验证伪似然与 benchmark accuracy 的相关性：在 GSM8K 的 100 个样本上，计算正确答案和错误答案的伪似然能量差异。如果差异不显著，整个方案的基础假设就不成立，应尽早止损。
- 考虑与 AdaReMask 方案合并：用 AdaReMask 的熵轨迹信号替代（或补充）伪似然能量函数，保留 EGIR 的退火调度和 MH 接受准则。这可能是兼顾理论优雅性和工程简洁性的最佳路径。
