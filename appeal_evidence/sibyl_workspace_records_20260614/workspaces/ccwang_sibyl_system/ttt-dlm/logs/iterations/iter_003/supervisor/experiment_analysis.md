# 实验结果分析

## 实验结果概要

Countdown-500（3 seeds）已完成 4/7 方法，核心方法 DTA 尚未出结果：

| 方法 | 均值准确率 | 与 vanilla 对比 |
|------|-----------|----------------|
| Vanilla | 4.7% (±0.6%) | 基线 |
| ReMDM-conf | 4.4% (±1.0%) | -0.3pp（略低于基线） |
| RCR | 5.7% (±0.6%) | +1.0pp（微弱提升） |
| **DMI** | **9.3% (±1.4%)** | **+4.6pp（~2x 提升）** |
| SCP | ~8.4%（interim, 150/500） | ~+3.7pp（中期数据） |
| DTA | pending | — |
| DTA+ReMDM | pending | — |

GSM8K-1319：Vanilla 29.6%（接近 Dream 论文报告值），ReMDM-conf 25.1%（仅 27% 进度，且低于基线）。

关键发现：
1. DMI（embedding 级跨步信息注入）以几乎零开销（~1.05x）实现了 ~2x 准确率提升
2. 纯 remasking 方法（ReMDM-conf, RCR）在 Countdown 上无实质改善
3. SCP 中期结果与 DMI 接近（~8.4% vs 9.3%），但计算开销远高
4. DTA 核心方法尚无任何实验数据

## 各方观点总结

- **乐观者**：DMI 的 ~2x 提升已构成论文核心贡献；remasking 的集体失败验证了"信息孤岛"假设；即使 DTA 失败，DMI + Correction Precision 框架 + 系统性负面结果已足够支撑高质量论文（ICLR/NeurIPS 级）。DTA 在 MBPP pilot 上 +12.5pp 暗示代码生成可能是其甜蜜点。论文叙事可重新定位为"跨步信息传递对 DLM 推理的关键作用"。

- **怀疑论者**：当前证据严重不完整——DTA 零数据，所有结论都是推测。DMI 的统计显著性未经 McNemar 检验验证；seed 间变异性较大（7.8%-10.6%，是 vanilla 的 2.3 倍）；存在替代解释（隐式温度调节、增加输入信息量）未被排除。Countdown 基线（4.7%）与 Dream 论文报告（16.0%）差距悬殊，可能指向评估 pipeline 问题。多重比较未做 Bonferroni 校正。核心建议：在 DTA 结果出来前停止任何乐观推测。

- **战略家**：项目处于战略有利位置——DMI 已提供"下行保护"安全网，论文可发表概率 ~90%，顶会概率 ~60%。DTA 完成仅需 ~20h 墙钟 GPU 自动运行，边际成本极低。根据 DTA 结果的 4 种场景已有明确论文定位策略。强烈建议 PROCEED，唯一触发 Pivot 的条件是 DTA 系统性低于 vanilla 且诊断确认是方法本身的根本局限。

## 分析

### 1. 方法可行性

DMI 作为 Level 1 消融基线的成功，实证验证了核心理论假设——跨步信息传递确实能改善 DLM 推理性能。DTA 的工程基础完全就绪：LoRA 范数稳定（<0.25，安全阈值 1.0）、跨模型兼容（Dream-7B + LLaDA-8B）、无退化（rep-3 甚至降低）、衰减机制按设计工作。但 DTA 在准确率上的实际效果完全未知。怀疑论者关于 ELBO 单调性在非凸 loss landscape 下仅局部成立的批评有理论依据，但不构成致命问题——大多数深度学习理论都在局部假设下成立。

**评估：工程可行性已充分验证，效果待定但基础扎实。**

### 2. 性能表现

已有结果中，DMI 是唯一展现实质性提升的方法（+4.6pp，~2x）。需注意的风险点：(a) 缺乏 McNemar p 值和 Bootstrap CI；(b) seed 间变异性较高（1.4% vs vanilla 的 0.6%）；(c) 绝对提升仅 4.6pp。然而，在 3 个独立 seed 上方向一致（7.8%/9.6%/10.6%），且效应量在 500 样本上应有足够统计功效——最终 p 值极可能显著。

SCP 中期结果（~8.4%）与 DMI 接近但计算开销高得多（~12x vs ~1.05x），这本身是一个重要发现。ReMDM-conf（4.4%）和 RCR（5.7%）基本无效，与 proposal 预测一致。

DTA 本身无任何性能数据——这是决策的核心不确定性。

**评估：DMI 结果方向明确；DTA 完全空白是最大悬念。**

### 3. 改进空间

无论 DTA 结果如何，都有清晰的改进路径：
- DTA 有效 → 超参数优化（学习率、LoRA rank、更新频率）、DTA+DMI 组合、跨任务泛化
- DTA 无效 → DMI 变体（注入 hidden states/attention）、DMI+SCP 组合、Correction Precision 框架推广
- DTA 在 MBPP pilot +12.5pp 和 LLaDA GSM8K 上 DTA+ReMDM 的 +12.4pp 组合互补性暗示任务特异性和方法正交性有探索空间

**评估：改进路径充裕，不存在死胡同。**

### 4. 时间成本

完成 DTA 仅需 ~20h 墙钟（3 seeds 可并行，GPU 自动运行，无需人力介入）。已有大量完成的消融数据（task_6a-8c）、诊断分析、跨模型验证——这些数据无论 DTA 结果如何都可直接用于论文。相比之下，pivot 到新方向意味着放弃已有的完整实验框架和数据，启动成本极高。

**评估：继续的边际成本远低于 pivot。**

### 5. 怀疑论者的批评是否致命？

逐项审查怀疑论者的核心关切：

- **Countdown 基线差距（4.7% vs 论文 16.0%）**：最值得关注的问题。但可能源于 prompt 格式、样本集、去噪步数等差异（Dream 论文未公布评估细节）。所有方法使用同一 pipeline，**相对比较仍然有效**。需确认原因但不影响 PROCEED 决策。

- **DMI 替代解释（隐式温度调节/增加输入信息量）**：合理关切，但可通过随机 logit 注入对照实验排除。这是论文完善阶段的工作，不影响当前优先级排序。

- **统计检验缺失**：确需补充 McNemar + Bootstrap CI + Bonferroni 校正。但这是结果整合阶段的标准操作，DMI 在 500 样本上的 4.6pp 效应量大概率通过校正后仍显著。

- **DTA 零数据就做乐观推测**：合理批评，但这恰恰是 PROCEED 而非 PIVOT 的理由——核心假设尚未验证就放弃，在时间成本如此低的情况下是不合理的。

**评估：所有批评均可在后续工作中解决，无致命阻断因素。**

## 决策理由

1. **核心方法未验证，放弃过早**：DTA 是论文的核心贡献，工程完全就绪但尚无任何结果。在核心假设验证前 pivot 等于在工程风险已消除、理论框架完整的情况下放弃一个仅差 ~20h GPU 时间就能出结果的方案——这在任何研究决策框架下都是不合理的。

2. **DMI 安全网已建立，下行风险可控**：即使 DTA 最终失败（场景 C/D），DMI 的 ~2x 提升 + remasking 的系统性失败 + 信息增强谱系消融 + Correction Precision 框架，已构成可发表的论文核心。战略家评估论文可发表概率 ~90%，我同意这个判断。真正需要 pivot 的概率 <5%。

3. **边际成本极低**：DTA 完成仅需 ~20h 墙钟 GPU 自动运行。已完成的消融/诊断/跨模型数据无论 DTA 结果如何都有价值。继续的机会成本几乎为零。

4. **四种场景均有出路**：DTA 大幅领先（→ NeurIPS 方法论文）、DTA 与 DMI 持平（→ ICLR 信息传递论文）、DTA 低于 DMI（→ NeurIPS/EMNLP DMI 发现论文）、DTA 全面失败（→ DMI 保底）。每种场景都有明确的论文定位和投稿策略。

5. **怀疑论者的关切可寻址**：统计检验、替代解释排除、基线校准——这些都是论文完善阶段的标准工作，不构成方向性阻碍。

DECISION: PROCEED
