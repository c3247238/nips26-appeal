# Lessons from This Iteration

## Must Improve

1. **死特征比率灾难性（89-99%）**: 所有训练 SAE 的死特征比率过高，可能使训练-SAE 实验分支完全无效。下一轮必须：
   - 增加训练 token 至 10M+（当前仅 1M）
   - 降低学习率至 1e-4（当前 1e-3）
   - 监控训练过程中的死特征比率
   - 如果无法降至 50% 以下，排除训练-SAE 实验

2. **架构比较严重混杂**: JumpReLU（Gemma 数据预训练，d_SAE=24,576）vs TopK（OpenWebText 从头训练，d_SAE=16,384）。4 倍碰撞率差异无法归因于架构。下一轮必须：
   - 在相同数据、相同字典大小下重新训练两种架构
   - 或者将架构比较降级为观察性发现，不作为核心贡献

3. **统计功效严重不足**: n=5-6 的数据点无法检测中等效应，所有 p 值 suspiciously 均匀 (~0.87)。下一轮必须：
   - 所有关键实验运行至少 3 个种子
   - 报告置信区间而非仅 p 值
   - 避免事后功效分析

4. **代理指标未验证**: "碰撞率"不等于 Chanin et al. 的"吸收"。论文建立在未验证的代理上。下一轮必须：
   - 在至少一个层次结构（元音 -> {a,e,i,o,u}）上实现真实吸收检测
   - 报告碰撞率与真实吸收率的相关性
   - 如果相关性弱，将 CAAB 重命名为"Cross-Architecture Collision Benchmark"

5. **论文范围过度扩张**: H1-H4 全部 inconclusive，但论文仍将其作为核心贡献。Result Debate 正确识别 UAD 是唯一稳健贡献。下一轮必须：
   - 将论文重新框定为围绕 UAD + DFDA
   - 将 H1-H4 降级为补充材料或未来工作

## Watch Out

1. **Pilot-full 差异**: Pilot（d_SAE=3,072）显示 30.8% 碰撞率，Full（d_SAE=16,384）显示 3.8%——8 倍差异。字典大小对碰撞率有巨大影响，必须在论文中讨论。

2. **Abstract 过度宣称**: "11.1% MSE improvement" 省略了"per-pair residual"限定词，显著夸大了贡献。所有 Abstract 声明必须与正文完全匹配。

3. **术语漂移**: "吸收"vs"碰撞"的混淆已大部分修复，但标题仍使用"Absorption"。需确保全文术语一致。

4. **特异性分数是数值伪影**: mean specificity 6.56e8 由除零驱动，不应报告。需使用有界指标（如 clip 到 100）。

5. **Gemma-2-2B 回退**: 计划中的 Gemma-2-2B 实验因 API 问题失败，但论文在 Abstract 和 Introduction 中仍提及。需明确所有结果仅基于 GPT-2 Small。
