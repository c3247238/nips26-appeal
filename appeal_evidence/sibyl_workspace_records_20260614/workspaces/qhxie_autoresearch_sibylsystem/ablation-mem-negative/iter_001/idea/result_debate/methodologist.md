# Methodologist 视角：方法论审查

## 实验设计问题

### 1. 样本量不足
- Pilot 实验使用 GPT-2 Small (L8, 1k samples)
- 全量实验样本量仍然偏小
- 建议：至少 3 个种子 (42, 123, 456)，每个 10k+ tokens

### 2. 控制变量不足
- 不同架构使用不同的训练配置
- 没有控制字典大小、稀疏度等混杂变量
- 建议：固定字典大小和 L0，只变架构

### 3. 评估指标问题
- "碰撞率"定义不明确：是共现频率？余弦相似度？激活重叠？
- 没有报告置信区间或 bootstrap 估计
- 建议：使用 Chanin et al. 的标准吸收检测协议

### 4. 复现性问题
- SAELens v6.39.0 API 兼容性问题导致 JumpReLU 加载失败
- 预训练 SAE 的加载不一致
- 建议：固定版本，提供完整环境配置

## 建议改进

1. 增加样本量到统计功效 ≥ 0.8
2. 使用标准化吸收检测协议
3. 控制混杂变量
4. 报告效应量和置信区间
