1. **循环 GT 定义必须重构**: "碰撞率验证真实吸收率"的叙事框架在 3 个迭代中持续存在，但审查者已明确识别其循环性。下一轮必须：
   - 将"碰撞率作为代理指标"重新框定为"碰撞率作为操作化定义的可靠性度量"
   - 在 Methods 中明确区分"操作化定义"和"独立验证"
   - 添加段落讨论 feature 11746 主导所有 26 个字母的问题

2. **数据-声明匹配必须零容忍**: Section 4.3 中 numbers r=0.843（实际 0.598）和 punctuation r=0.891（实际 0.693）的数据不匹配是严重错误。下一轮必须：
   - 写作审查阶段强制要求每个数字与源 JSON 文件交叉验证
   - 任何无法追溯到源数据的数字必须标记为"待验证"或删除
   - 在写作 pipeline 中添加自动化的数字一致性检查

3. **图表生成必须与数据收集同等优先级**: 连续 3 个迭代引用不存在的图表。下一轮必须：
   - 在 plan 阶段将每个图表/表格作为独立任务分配
   - 图表生成任务阻塞写作任务的完成
   - 使用 matplotlib/seaborn 从 JSON 结果自动生成

4. **编造声明零容忍**: "手动检查 50 个假阳性"在结果文件中无任何证据。下一轮必须：
   - 任何声明必须有对应的 results JSON 文件支持
   - 写作审查阶段专门检查"无源数据声明"
   - 宁可删除弱声明，绝不编造

5. **K-means 成功必须分析**: 85.7% 召回被草率否定是分析上的失职。下一轮必须：
   - 分析 K-means 为何成功（是否按非共现属性分组？）
   - 探索后处理阈值是否能改善精度
   - 将此分析作为建设性贡献的一部分

## Watch Out

1. **普遍声明陷阱**: 每次迭代都倾向于从有限证据（单一 SAE、单一层次类型、单一例子）泛化到普遍结论。必须显式限定范围："对于已测试的 token-不相交层次"。

2. **F1 恒等式误导**: UAD F1 = 同簇随机 F1 = 0.00048 是数学恒等式（相同的 TP/FP/FN），不是统计发现。表述为"两者都恰好检测到 1 个真正例"而非"统计上不可区分"。

3. **术语一致性**: "Spearman r" vs "Spearman rho"、"false positive rate" vs "1 - precision" 等术语漂移会损害可信度。建立术语检查清单。

4. **影响声明过度**: "阻止社区走向死胡同"、"放弃共现方法"等措辞强于证据支持。使用"对此特定设置"、"建议探索替代方案"等限定语。

5. **自对 [24189, 24189] 的 GT 计数**: 7 个 GT 对中包含一个自对，实际不同对为 6 个。必须澄清计数方式并确保精度/召回分母一致。

6. **消融中过滤步骤无效果**: "no_dead_filter" 和 "no_phi" 与完整 UAD 产生完全相同的结果，暗示这些过滤步骤是 no-op。必须报告实际过滤了多少特征。
