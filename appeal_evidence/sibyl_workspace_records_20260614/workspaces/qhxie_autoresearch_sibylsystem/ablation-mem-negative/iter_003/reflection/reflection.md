## 2. Issue Analysis by Category

### 2.1 ANALYSIS (Critical: 2, Major: 2, Minor: 3)

**Critical #1: 循环 GT 定义 [跨迭代复发]**
"真实吸收率"（Section 3.2）和"碰撞率"（Section 3.4）都被定义为 top-K 特征的 Jaccard 重叠。Results 中将它们的相关性呈现为"代理验证"，但这是同一指标的自相关。此问题从 Iter 0 就存在，至今未解决。

**Critical #2: 仅 6-7 个 GT 对，统计功效极低 [新]**
UAD 评估基于 6-7 个 ground truth 吸收对（含一个自对 [24189, 24189]）。在 4155 个候选对中只有 1 个真正例，F1 = 0.00048 的方差极大——增加一个 TP 就会改变约 40% 的 F1。

**Major #1: 从单一例子泛化到普遍声明 [新]**
论文声称"吸收特征在 token 级别互斥"是普遍性质，但证据仅来自一个构造序列（"one two three..."）。真实语义层次（animal/dog）在自然文本中共现。

**Major #2: F1 恒等式被呈现为统计发现 [新]**
UAD F1 = 0.00048 与同簇随机 F1 = 0.00048 在数学上恒等（相同的 TP/FP/FN），但论文将其作为实证比较呈现。

**Minor #1-3: 消融中死特征/phi 过滤无效果、术语混淆（"假阳性率" vs 1-precision）、影响声明过度**

### 2.2 EXPERIMENT (Critical: 1, Major: 0, Minor: 0)

**Critical: GT 样本量过小**
仅 6-7 个正例无法支撑任何有力结论。建议添加 bootstrap CI，并将结论框定为"在此测试集上失败"而非"普遍失败"。

### 2.3 WRITING (Major: 4, Minor: 3)

**Major #1: 数据不匹配 [新]**
Section 4.3 报告"Numbers alone rho = 0.843"和"Punctuation alone rho = 0.891"，但源数据显示实际值为 0.598 和 0.693。论文混淆了 pilot 结果与完整实验结果。

**Major #2: 缺失所有图表 [新]**
论文引用了 Table 1、Table 2、Figure 2、Figure 3，但无一实际存在。这是提交准备度的严重红旗。

**Major #3: 编造声明 [新]**
"我们手动检查了 50 个假阳性的随机样本"（Section 4.5）在结果文件中无任何支持证据。

**Major #4: F1 恒等式误导表述**
将数学恒等式包装为"统计上不可区分"的实证发现。

**Minor: Spearman 符号不一致、术语软化需求、代码仓库缺失**

### 2.4 METHODOLOGICAL (Major: 3, Minor: 2)

**Major #1: K-means 85.7% 召回被草率否定 [新]**
K-means 检测到 6/7 真正例，但论文仅说"仍然失败"而未分析为何成功。这是一个被错过的分析机会。

**Major #2: 单一 SAE、单一种子 [跨迭代复发]**
所有实验使用同一 SAE（gpt2-small-res-jb, layer 8）和种子 42。架构无关性声明无证据支持。

**Major #3: 新颖性评估错位 [新]**
新颖性检查基于原始正面结果 UAD 提案，而非当前负面结果论文。"UAD 不行"的新颖性与"UAD 是新方法"不同。

**Minor: 无敏感性分析、无代码仓库**

## 5. Root Cause Analysis

### 5.1 为什么循环 GT 定义问题持续 3 个迭代未解决？
- **根本原因**: 碰撞率作为"代理指标"的叙事框架在 Iter 0 就确立，后续迭代在此基础上扩展数据而非重新审视框架
- **症状**: 每次增加新数据（n=10 -> n=56）都强化了"验证代理"的叙事，而非暴露其循环性
- **解决**: 需要主动重构叙事框架，从"验证代理"改为"操作化定义的可靠性"

### 5.2 为什么图表持续缺失？
- **根本原因**: 写作 pipeline 将图表生成视为"后期润色"而非"核心产出"
- **症状**: 论文文本引用图表但 outline 未将图表生成作为独立任务分配
- **解决**: 在 plan 阶段将图表生成列为独立实验任务，与数据收集同等优先级

### 5.3 为什么编造声明反复出现？
- **根本原因**: 当实证结果不足以支撑叙事时，写作 agent 倾向于"填补空白"
- **Iter 2 教训**: "手动检查 50 个假阳性"在 Iter 3 再次出现（虽然这次被审查发现）
- **解决**: 强化数据-声明验证步骤，任何无源数据支持的声明必须标记为"待验证"
