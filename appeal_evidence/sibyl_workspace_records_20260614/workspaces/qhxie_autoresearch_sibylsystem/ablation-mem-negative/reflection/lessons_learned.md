# Lessons from This Iteration

## Must Improve

1. **随机基线是解释性实验的必需品**: F1=0.725 在没有随机基线的情况下完全不可解释。从 C(500,2)=124,750 可能对中选 51 对，随机基线计算量极小（<1 分钟）却至关重要。任何声称"优于随机"的实验都必须实际计算随机基线，不能跳过或降级为"补充材料"。

2. **区分"确定性"与"鲁棒性"**: 多种子一致性（std=0.000）在固定 SAE 权重下是 trivially achievable 的确定性结果，不是鲁棒性。Section 标题、Abstract 和正文必须一致使用准确术语。在固定权重 + 足够样本的条件下，确定性是预期行为，不应作为贡献声称。

3. **DFDA 度量同义反复**: 在 child-dominant 位置测量 MSE 改进，MLP 学习预测近零值，改进在数学上被保证。这不是吸收恢复，是度量设计缺陷。任何补偿/恢复方法的评估必须在 parent-positive 示例上进行（父特征应该激活的输入）。

4. **跨层结果必须逐层报告**: Layer 4（F1=0.432）低于阈值是真实结果，不应被均值掩盖。逐层报告 + 诚实讨论失败原因，比掩盖失败更能建立评审信任。

5. **消融实验是方法论文的门槛**: 任何声称"我们使用 X 方法"的论文都必须证明 X 的选择优于替代方案。k-means vs HAC、phi vs raw、不同 cluster 数——每个消融仅需 ~10 秒，却是方法严谨性的基础。

6. **结构缺陷是投稿阻塞项**: 缺失 E4、表格编号不一致、Figure 1 缺失——这些问题在评审眼中比实验缺陷更显眼，因为它们表明作者缺乏基本的论文校对能力。Outline 与实际 paper 必须保持同步。

## Watch Out

1. **"Any SAE, any corpus" 过度泛化**: 单模型、单层、单概念域的验证结果不能支撑如此广泛的适用性声称。Table 1 中的此类声称会被评审直接标记为 overclaiming。

2. **Phi 值完全相同需验证**: Table 4 中 5 个不同 pair 的 phi 值完全相同到浮点精度，这在统计上极不可能。要么有合理的聚类结构解释，要么是数据错误。

3. **内部术语泄漏到论文**: "PARTIAL_PASS" 等内部验证语言不应出现在论文草稿中。写作 agent 需要明确的"禁止术语清单"。

4. **引用不当削弱可信度**: Elhage et al. (2022) 关于 superposition 被用于支持层-wise hierarchy 声称，这是 weak citation。每个引用必须直接支持其声称。

5. **DFDA caveat 位置**: 强烈的免责声明被埋在段落中间，快速浏览的评审可能错过。重要限制应放在 section 开头，使用醒目标记（如 **Preliminary Result**）。

6. **计划时间与实际时间差异**: 所有实验实际运行时间（~1 秒）远低于计划时间（15-30 分钟），说明计算需求被严重高估。这本身不是错误，但意味着更多实验可以在相同预算内完成。

## Keep Doing (success patterns)

1. **诚实报告局限性**: DFDA caveat（"near-100% improvement is artifactual"）和 determinism vs robustness 区分是 exemplary 的科学写作。这种诚实建立评审信任，是论文的最强资产。

2. **可验证的数据完整性**: 每个数字都与源 JSON 文件完全匹配。写作评审验证了 F1=0.725、precision=0.569、recall=1.0 等所有关键数字。继续保持这一标准。

3. **成功 Pivot 能力**: 从 Iteration 0 失败的"跨架构碰撞基准"重新框定为聚焦的 UAD 方法论文，展示了系统识别核心贡献并调整方向的能力。

4. **清晰的算法描述**: 六步 UAD pipeline 足够精确以重新实现。方法部分的清晰度是论文的可复现性基础。

5. **完美 recall 的诚实定位**: "screening tool, not classifier" 框架诚实管理期望，将 43% 假阳性率从弱点转化为诚实的范围界定。

6. **零失败实验执行**: 所有 6 个任务成功完成，技术基础设施稳健。
