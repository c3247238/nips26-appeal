## 根本原因分析

**根因 1：流水线封锁机制缺失**
action_plan.json 有 blocking=true 字段，但写作 Agent 不检查它。这是技术问题，可以通过在编排器中添加一个前置验证步骤来解决。

**根因 2：写作 prompt 中对理论-证据对齐的检查不足**
写作 Agent 在写 Introduction 时复制了"Proposition 1 predicts..."的语言，而不检查 B1_decoder_geometry.json 是否支持这一预测。写作 prompt 应包含明确指令：在使用任何定理作为实证预测时，先查验对应实验的 AUROC 方向。

**根因 3：标签集来源未在计划阶段统一**
三套不同标签集（n=18 精确标签、n=50 代理标签、n=63 B1 阈值标签）来自不同实验阶段，在计划阶段未被识别为需要协调的问题。下次迭代的计划任务应包含"标签集统一审计"作为写作前置条件。
