# 方法论分析：实验设计的严谨性评估

## 总体评估

Pilot 实验的设计**基本合理**但存在多个需要改进的方面。核心发现（吸收率降低）在 methodologically 上是可信的，但需要额外的控制实验来排除替代解释。

## 实验设计审查

### 1. 吸收率指标的设计与验证

**指标定义：**
- absorption_rate = mean(max_j(coactivation(i,j) > threshold))
- coactivation 基于神经元在验证集上的共激活频率

**优点：**
- 指标直观：测量"一个特征被激活时，其他特征也被激活的程度"
- 成功区分了 trained/random/sparse 条件
- 与理论概念（polysemanticity/superposition）有明确联系

**问题：**
- 阈值选择（默认 0.01）的敏感性未测试
- 是否应在激活值上标准化？（当前没有）
- 未报告指标的内部一致性（如 split-half reliability）

**建议：**
- 进行阈值敏感性分析（threshold ∈ {0.001, 0.01, 0.1}）
- 报告吸收率的分布而不仅是均值

### 2. 单一种子的统计功效

**现状：** pilot 只有 seed=42

**计算所需样本量：**
- 效应量 d = 5.51（来自 iter_005 的 5 种子数据）
- α = 0.05, power = 0.80
- 每组所需 n ≈ 2-3（因为效应量极大）

**结论：** 即使只有 3-5 种子，统计功效也足够。但单种子 pilot 无法估计方差。

### 3. 架构比较的公平性

**当前比较：**

| 变体 | L0 | 稀疏化机制 | 吸收率 |
|------|-----|-----------|--------|
| Baseline | 1047 | L1 penalty | 0.226 |
| TopK | 50 | Hard threshold | 0.033 |
| MultiScale | 50 | Multi-scale decomposition | 0.042 |

**混淆因素：**
- 比较同时改变了"架构"和"稀疏性水平"两个变量
- 无法区分"TopK 架构有效"vs"L0=50 有效"

**缺失的控制：**
- L0 匹配的 Baseline（通过调整 λ 使 Baseline 也达到 L0≈50）
- 这个控制是**至关重要**的，它将决定故事是"架构很重要"还是"稀疏性很重要"

### 4. 死神经元的影响

**数学分析：**

吸收率 = (1/N) Σ_i max_j c(i,j) > threshold

如果神经元 j 死亡（永不激活），则对所有 i，c(i,j) = 0。

因此死神经元不贡献于 max_j c(i,j)（因为 0 < threshold）。

**关键问题：** 死神经元减少了"候选共激活者"的池子。如果有 83% 的神经元死亡，那么每个活跃神经元只能在剩余的 17% 中寻找共激活伙伴。这**机械性地降低了**找到高共激活伙伴的概率。

**验证方法：**
- 计算仅活跃神经元的吸收率：只考虑在验证集中至少激活一次的神经元
- 如果活跃-神经元-only 吸收率仍然显著降低，则结论成立
- 如果差异消失或大幅缩小，则死神经元是主要驱动因素

### 5. 重构质量的评估

**发现：** 所有模型的 explained_variance 为负

**解释：**
- 合成数据可能具有特殊的统计特性
- SAE 的隐藏维度（2048）相对于输入维度可能不足
- 训练时间可能不够

**影响评估：**
- 如果 SAE 不能重构输入，它是否在"学习有意义的特征"？
- 但 random control 的重构更差（-118.63 vs -0.93），说明训练确实在学习 something
- 吸收率的降低发生在 trained 模型之间，而非 trained vs random 之间，所以相对比较仍然有效

### 6. Random Control 的设计

**当前设计：** 使用随机初始化的权重，不训练

**优点：**
- 简单、可复现
- 提供了吸收率的"上限"参考

**潜在问题：**
- 随机权重可能产生与 trained 模型完全不同的激活分布
- 可以考虑更精细的控制：如训练相同步数但用随机梯度

## 统计检验建议

对于完整实验，建议使用：

1. **主效应检验：** 单因素 ANOVA（Baseline vs TopK vs MultiScale vs L0-match）
2. **成对比较：** Tukey HSD post-hoc
3. **效应量报告：** Cohen's d 及 95% CI
4. **非参数验证：** Kruskal-Wallis（以防分布不正态）

## 威胁评估

| 威胁类型 | 描述 | 严重程度 | 缓解策略 |
|----------|------|----------|----------|
| 内部效度 | L0 混淆 | 高 | 添加 L0 匹配控制 |
| 内部效度 | 死神经元人为因素 | 高 | 报告活跃-神经元-only 分析 |
| 外部效度 | 仅合成数据 | 中 | Gemma Scope 验证 |
| 构念效度 | MCC 不敏感 | 中 | 已识别，使用吸收率替代 |
| 统计结论效度 | 单种子 | 中 | 5 种子完整实验 |

## 结论

**方法论评分：6.5/10**

- 核心指标设计合理 (+)
- 架构比较存在混淆变量 (-)
- 死神经元问题需要认真处理 (-)
- 合成数据的限制需要真实 LLM 验证 (-)

**最优先的补充实验：**
1. L0 匹配 Baseline 控制
2. 活跃-神经元-only 吸收率分析
3. 5 种子重复
