# 怀疑论者分析：对 Pilot 结果的严格审视

## 核心观点

虽然结果看起来积极，但存在**多个严重的 methodological 和方法论问题**，在庆祝之前必须正视。

## 关键质疑

### 1. 单一种子 = 无统计显著性

Pilot 只用了 seed=42。这意味着：
- 我们不知道 85.5% 的降低是稳定的还是 cherry-picked
- 随机种子的选择可能恰好有利于 TopK
- 没有标准差、没有置信区间
- **这不是科学发现，这是一个数据点**

之前的完整实验（iter_005，5 种子）虽然支持同一方向，但那是不同迭代的数据，可能来自不同的实验设置。

### 2. MCC 的彻底失效

| 变体 | MCC |
|------|-----|
| Baseline | 0.216 |
| TopK | 0.215 |
| MultiScale | 0.219 |
| Random | 0.221 |

MCC 的变异仅为 ~0.006，甚至 Random control 的 MCC 与 trained 模型相同！这说明：
- **MCC 完全无法区分训练/随机/稀疏/非稀疏**
- 如果连最基本的特征恢复都无法测量，我们在测量什么？
- 吸收率可能是我们在一堆无效指标中找到的一个"看起来有效"的指标——confirmation bias 风险

### 3. 死神经元可能是吸收降低的人为因素

TopK 83.2% 死神经元率引发根本性问题：
- 如果 83% 的神经元不活跃，那么活跃神经元之间的"共激活"自然减少
- 吸收率 = mean(max coactivation > threshold)。如果大部分神经元永久关闭，分母变小，吸收率自然降低
- **这不是"TopK 减少了吸收"，而是"TopK 杀死了神经元，从而机械性地降低了共激活统计量"**
- 必须报告仅活跃神经元的吸收率才能排除这一解释

### 4. 负的解释方差

所有 trained 模型的 explained_variance 都是负的：
- Baseline: -0.934
- TopK: -0.403
- MultiScale: -0.335
- Random: -118.63

虽然 Random 更差（符合预期），但**所有 trained 模型都未能正重构输入**。负的 explained_variance 意味着 SAE 的重构比直接用均值预测还差。

这提出了根本问题：
- 如果一个 autoencoder 不能比均值更好地重构，它在学习什么？
- 吸收率的降低是否发生在"学习失败"的前提下？
- 这些结果在"无法正常工作的 SAE"上观察到的，对实际部署的 SAE 是否有意义？

### 5. 仅在合成数据上验证

SynthSAEBench-1k 是人工构造的数据集：
- 真实的 token 共现模式可能与合成数据完全不同
- 真实 LLM 的 SAE 面临的是高维、稀疏、长尾分布
- 结果是否能推广到 Gemma Scope 或 GPT-4 SAE？
- **在人工数据上的发现不等于在自然数据上的发现**

### 6. 架构比较的公平性问题

TopK 和 Baseline 是否在公平的条件下比较？
- TopK 使用 hard sparsity (L0=50)，Baseline 使用 soft sparsity (L0=1047)
- 如果 Baseline 也用 L0=50（通过调整 L1 penalty），吸收率会如何变化？
- 当前比较混淆了"架构差异"和"稀疏性水平差异"
- L0 匹配控制实验是必要的，但尚未运行

## 结论

**在以下问题得到回答之前，不能宣布胜利：**
1. 5 种子重复是否确认 85.5% 的降低？
2. 仅活跃神经元的吸收率是否仍然支持结论？
3. L0 匹配的 Baseline 是否仍然显示高吸收？
4. 真实 LLM SAE 上是否复现？
5. 重构质量是否可以接受？

**建议：谨慎推进，优先解决上述 methodological 问题。**
