# 修正主义者分析：调和矛盾与更新心智模型

## 识别出的矛盾

### 矛盾 1：TopK 是最优解还是病理解？

**表面发现：** TopK 吸收率最低（0.033），似乎是最优架构

**隐藏问题：**
- 83.2% 的死神经元率意味着模型使用了不到 17% 的容量
- 这是否是"好的稀疏性"还是"训练失败"？
- 如果 83% 的特征从未激活，我们是否真的有一个 1024 维的表示，还是一个 ~170 维的表示？

**调和尝试：**

两种解释框架：

**框架 A：TopK 是真正的胜利**
- 死神经元是稀疏优化的已知副作用
- 活跃的特征更少，因此更"专注"，更少吸收
- 83% 的死神经元可以透明地报告为 limitation

**框架 B：TopK 是人为的低吸收**
- 死神经元机械性地减少了共激活的候选池
- 如果只看活跃神经元，吸收率可能不这么低
- 需要验证：活跃-神经元-only 吸收率

**当前立场：** 倾向于框架 A，但必须在完整实验中验证框架 B。

### 矛盾 2：MCC 的完全失效

**发现：** MCC 在所有变体中几乎相同（0.214-0.221），包括 Random

**含义：**
- MCC 完全无法区分 trained 和 random
- 但我们声称 SAE "学习了有意义的特征"
- 如果连特征恢复都无法测量，特征"有意义"是什么意思？

**调和尝试：**

MCC 可能测量的是错误的东西：
- MCC 假设每个隐藏神经元对应一个 ground-truth 特征
- 但 SAE 的神经元可能进行分布式编码（多个神经元编码一个特征）
- 或者一个神经元编码特征的多个方面

因此：
- MCC 的低值**不意味着** SAE 没有学习结构
- 它意味着 SAE 的表示与简单的"一神经元一特征"假设不一致
- 吸收率捕获了另一种结构：**神经元间的依赖关系**

**更新后的心智模型：**
- SAE 学习结构，但不是"特征字典"形式
- 学习的是**条件激活模式**（哪些特征一起激活）
- 吸收率测量的是这种条件结构的"纠缠程度"

### 矛盾 3：负的解释方差

**发现：** 所有 trained 模型的 explained_variance < 0

**问题：** SAE 的重构比直接用均值预测还差

**调和尝试：**

可能的原因：
1. **合成数据的特殊性：** 如果输入具有零均值和单位方差，且 SAE 重构有偏，可能导致负值
2. **训练不足：** 如果训练时间不够，模型可能尚未收敛
3. **评估方法问题：** explained_variance 对异常值敏感

**关键观察：** Random 的 explained_variance 更差（-118 vs -0.93）
- 这说明训练**确实在改善**重构
- 只是改善后仍然低于基线

**对吸收率发现的影响：**
- 即使重构不完美，SAE 仍然学习到了**激活模式结构**
- 吸收率比较的是 trained 模型之间的**相对差异**
- Random 作为上限参考，其吸收率确实更高

### 矛盾 4：稀疏性是"原因"还是"相关性"

**主张：** 稀疏性降低吸收（因果）
**替代解释：** 稀疏性和吸收降低都是某个深层因素的共同结果

**可能的共同原因：**
- 更稀疏的激活 → 更少的特征竞争 → 更少的吸收
- 或者：某些架构同时产生稀疏激活和低吸收（但机制不同）

**区分方法：**
- L0 匹配控制实验是关键
- 如果 Baseline 强制 L0=50 后吸收率也降低 → 稀疏性是原因
- 如果 Baseline 强制 L0=50 后吸收率仍高 → 架构差异是原因

## 更新后的心智模型

基于 pilot 结果和矛盾调和，更新以下心智模型：

### 模型 v1.0（迭代前）
```
吸收 ← 架构选择
TopK 减少吸收 because TopK architecture is better
```

### 模型 v2.0（当前最佳猜测）
```
吸收 ← 稀疏性水平 ← 架构选择
TopK 减少吸收 because it enforces sparsity
Architecture affects absorption indirectly via sparsity
```

### 模型 v3.0（待验证）
```
吸收 ← f(稀疏性, 激活分布, 特征相关性)
稀疏性是主要因素，但激活分布的形状也有影响
不同架构可能产生相同稀疏性但不同吸收率
```

## 风险重评估

| 假设 | 之前信心 | 现在信心 | 变化原因 |
|------|----------|----------|----------|
| 稀疏性降低吸收 | 0.7 | 0.85 | Pilot 强力支持 |
| 架构不重要 | 0.3 | 0.6 | 需 L0 匹配控制验证 |
| 吸收率是好的指标 | 0.8 | 0.75 | MCC 失效提醒需谨慎 |
| 结果可推广到真实 LLM | 0.5 | 0.5 | 尚无证据 |

## 对下一步的建议

1. **最高优先级：** L0 匹配控制实验（区分稀疏性 vs 架构效应）
2. **高优先级：** 活跃-神经元-only 吸收率（排除死神经元人为因素）
3. **高优先级：** 真实 LLM 验证（外部效度）
4. **中优先级：** 理解为什么 MCC 失效（理论完整性）
5. **低优先级：** 改善重构质量（如果目标是发表而非实际部署）

## 结论

Pilot 结果支持核心方向，但多个矛盾提醒我们：
- **不要过早下因果结论**
- **控制实验是必须的**
- **透明报告 limitation 比隐藏更有价值**

当前最佳策略：**ADVANCE with caution** — 推进实验，但保持开放的心态，准备根据新证据修正理论。
