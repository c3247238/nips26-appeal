# Methodology: Synthetic-to-Real Transfer for Absorption Metric Validation (Round 3)

## Overview

This document details the experimental methodology for Round 3, following the PIVOT decision to Candidate C. After two rounds of pilot experiments on real LLM SAEs revealed fundamental measurement challenges (lack of ground-truth, confounded metrics, degenerate probe tasks), we switch to a synthetic-to-real validation framework.

## Core Problem Addressed

**Round 1-2 Finding**: Real SAE absorption metrics cannot be validated because ground-truth is unknown. Feature overlap between semantic categories is equally explained by:
- True mechanistic absorption
- Semantic similarity (polysemantic neurons)
- Superposition effects
- SAE training artifacts

**Round 3 Solution**: Create synthetic data with known hierarchical structure, train SAEs on it, and validate which metrics correctly identify absorption. Then transfer calibrated metrics to real SAEs with confidence intervals.

## Research Questions

**RQ1**: Which absorption metrics (Chanin et al., SAEBench, encoder-decoder cosine, conditional activation) accurately detect absorption on ground-truth synthetic data?

**RQ2**: How do metric accuracy, precision, and recall vary with:
- Parent-child co-occurrence frequency
- Feature sparsity level
- Dictionary size
- Noise level
- SAE architecture (Standard ReLU vs TopK)

**RQ3**: Do metrics calibrated on synthetic data transfer to real LLM SAEs, and how does the synthetic-to-real gap vary with model scale?

## Experimental Setup

### Phase 1: Synthetic Data Generation

#### Hierarchical Feature Generator

Define a semantic hierarchy with 3 levels:
- **L0 (Abstract)**: 10 parent concepts (e.g., animal, vehicle, food)
- **L1 (Category)**: 5 sub-concepts per parent (e.g., dog, cat, bird under animal)
- **L2 (Instance)**: 3 instances per sub-concept

**Total features**: 10 + 50 + 150 = 210 semantic features

#### Activation Generation Rules

For each sample, activations are generated by:

```
# Parent feature activation
if child_present:
    parent_activation = p_parent_given_child * base_activation + noise
else:
    parent_activation = p_parent_alone * base_activation + noise

# Child feature activation
child_activation = base_activation + noise

# Non-hierarchical features: random sparse activation
random_features = sparse_random_activation(sparsity_target)
```

**Controllable parameters**:
| Parameter | Range | Description |
|-----------|-------|-------------|
| `p_parent_given_child` | [0.3, 0.9] | Parent activation when child present |
| `p_parent_alone` | [0.0, 0.5] | Parent activation when child absent |
| `sparsity` | [0.01, 0.20] | Fraction of features active per sample |
| `noise_std` | [0.0, 0.3] | Gaussian noise on activations |
| `n_samples` | 10000 | Number of synthetic samples |
| `d_model` | 128 | Activation dimension |

**Ground-truth absorption label**:
A parent feature is "absorbed" if `p_parent_alone < 0.2 * p_parent_given_child`.

#### Synthetic Data Variants

Generate 3 complexity levels:
- **Simple**: 50 features, 2-level hierarchy, low noise
- **Medium**: 210 features, 3-level hierarchy, medium noise
- **Complex**: 500 features, 3-level hierarchy + distractors, high noise

### Phase 2: SAE Training on Synthetic Data

Train small SAEs with controlled configurations:

| Config | d_model | d_sae | Architecture | L1/TopK |
|--------|---------|-------|--------------|---------|
| C1 | 128 | 512 | Standard ReLU | L1=1e-3 |
| C2 | 128 | 1024 | Standard ReLU | L1=5e-4 |
| C3 | 128 | 2048 | Standard ReLU | L1=2e-4 |
| C4 | 128 | 512 | TopK | k=10 |
| C5 | 128 | 1024 | TopK | k=20 |

**Training details**:
- Optimizer: AdamW, lr=1e-3
- Batch size: 256
- Steps: 5000 (quick convergence on synthetic data)
- Validation split: 20%
- Seed: 42

**Estimated time per SAE**: 3-5 minutes on single GPU

### Phase 3: Metric Validation

For each trained SAE, compute ground-truth absorption labels and apply candidate metrics:

#### Metric 1: Chanin et al. Ablation-Based Absorption

```python
# For each parent feature f_p and child feature f_c:
# 1. Measure parent activation with child active
act_with = sae_features[f_p] when f_c active

# 2. Zero out child feature, measure parent activation
act_without = sae_features[f_p] when f_c zeroed

# 3. Absorption score = 1 - act_without / act_with
absorption_chanin = 1 - act_without / act_with
```

#### Metric 2: SAEBench Absorption Metric

Use SAEBench's built-in absorption metric implementation (if available) or replicate their approach.

#### Metric 3: Encoder-Decoder Cosine Similarity

```python
cosine_sim[i] = (W_enc[:, i] . W_dec[i, :]) / (||W_enc|| * ||W_dec||)
```

Hypothesis: Absorbed features have lower cosine similarity.

#### Metric 4: Conditional Activation Probability

```python
P_on = P(act_parent > thresh | act_child > thresh)
P_off = P(act_parent > thresh | act_child < thresh)
ratio = P_off / P_on
```

Hypothesis: Absorbed pairs have ratio < 0.5.

#### Validation Protocol

For each metric:
1. Compute metric scores for all parent-child pairs
2. Compare to ground-truth absorption labels
3. Calculate: ROC-AUC, precision, recall, F1, calibration curve
4. Identify optimal threshold via Youden's J statistic

### Phase 4: Parameter Sweep

Systematically vary synthetic data parameters to characterize metric robustness:

| Sweep | Parameter | Values | Fixed |
|-------|-----------|--------|-------|
| S1 | Co-occurrence (p_parent_given_child) | [0.3, 0.5, 0.7, 0.9] | sparsity=0.05, noise=0.1 |
| S2 | Sparsity | [0.01, 0.05, 0.10, 0.20] | cooc=0.7, noise=0.1 |
| S3 | Noise | [0.0, 0.1, 0.2, 0.3] | cooc=0.7, sparsity=0.05 |
| S4 | Dictionary size | [512, 1024, 2048] | cooc=0.7, sparsity=0.05, noise=0.1 |

**Total SAEs to train**: 4 + 4 + 4 + 3 = 15 (plus 5 baseline configs = 20)
**Estimated total time**: ~2 hours (parallelizable across GPUs)

### Phase 5: Transfer to Real SAEs

Apply calibrated metrics to real LLM SAEs:

#### Real SAE Targets

| Model | SAE Source | Layers | Widths |
|-------|-----------|--------|--------|
| GPT-2-small | SAELens (gpt2-small-res-jb) | 0, 4, 8 | 24576 (single width) |
| Gemma-2-2B | GemmaScope (if accessible) | 8 | 16k, 65k |

#### Transfer Protocol

1. **Identify hierarchical features in real SAEs**:
   - Use Neuronpedia or manual inspection to find parent-like features (broad concepts)
   - Find child-like features (specific instances)
   - Validate parent-child relationship via co-occurrence on text corpus

2. **Apply calibrated metrics**:
   - Use thresholds learned from synthetic data
   - Compute metric scores for identified pairs
   - Report metric values with confidence intervals (from synthetic calibration curves)

3. **Cross-metric consistency check**:
   - Compare predictions across metrics
   - Flag pairs with high disagreement for manual inspection

4. **Measure synthetic-to-real gap**:
   - Compare metric distributions between synthetic and real SAEs
   - Correlate gap with model scale, SAE width, layer depth

## Baselines

1. **Random classifier**: AUC = 0.5 baseline
2. **Oracle (ground-truth)**: Perfect classification for synthetic data only
3. **Feature overlap (Round 1-2 metric)**: Compare to our previous semantic overlap metric to quantify improvement

## Evaluation Criteria

### For Synthetic Validation

| Metric | Target | Description |
|--------|--------|-------------|
| ROC-AUC | > 0.85 | Classification accuracy |
| Calibration error | < 0.10 | Reliability of probability estimates |
| Cross-parameter stability | CV < 15% | Consistency across parameter sweeps |

### For Real SAE Transfer

| Metric | Target | Description |
|--------|--------|-------------|
| Cross-metric agreement | > 70% | Multiple metrics agree on absorption status |
| Confidence interval coverage | > 80% | True absorption rate within predicted CI |
| Synthetic-to-real correlation | r > 0.5 | Metric rankings consistent across domains |

## Risk Mitigation

| Risk | Likelihood | Mitigation |
|------|------------|------------|
| Synthetic data too simple | Medium | Test 3 complexity levels; include distractor features |
| Real SAEs lack clear hierarchies | High | Use multiple hierarchy types (WordNet, spelling, functional); report coverage |
| Gemma-2-2B access fails | Medium | Fallback to GPT-2-small only; use multiple layers as proxy for width variation |
| Metric calibration unstable | Low | Use 5-fold cross-validation on synthetic data; report variance |
| SAE training on synthetic data fails | Low | Validate reconstruction MSE < 5% before metric evaluation |
