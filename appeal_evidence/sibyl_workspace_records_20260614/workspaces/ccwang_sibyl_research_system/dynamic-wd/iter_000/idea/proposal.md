## 三个强制改进（优先级 1A-1C）

### 改进 1A：多 seed 实验（NeurIPS 标准）【时间：2.5h】

**现状问题**：
- 所有对比仅用 seed=42
- 边际差异：AADWD Conservative vs. Fixed WD = -0.17%，Random vs. AADWD Aggressive = 0.01%
- 这些差异小于典型的训练噪声，无法排除随机波动假设

**行动**：
```
运行表1（7个方法）× 5个seed（42, 123, 456, 789, 1010）× 200-epoch
结果：每个单元格 mean ± 2σ (95% CI)
关键验证：
  - "AADWD保守比固定WD差0.17%"：95% CI是否明确包含负值？
  - "随机WD略优于AADWD"的0.01%差异：是否统计显著？
```

**为什么关键**：
- NeurIPS/ICML 明确要求多 seed + 置信区间
- 大效应结果（解耦collapse：92%→10%）已充分稳健，无需改变
- 这一步直接影响审稿人的第一印象 ("作者知道标准吗？")

**优先级标签**：**1A - MANDATORY**

---

### 改进 1B：解耦实验的 c 值一致性【时间：0.5h】

**现状问题**：
- 主实验：aggressive c=2.5, conservative c=0.005
- 解耦实验（"完全去耦"）：aggressive c=0.25, conservative c=0.0005（各小 10 倍）
- 审稿人可能质疑：collapse 是因为耦合被破坏，还是因为 c 值太大？

**行动**：
```
重新运行解耦变体，使用与主实验相同的 c 值：
  - Aggressive: λ_t = c=2.5 · γ_t · δ_t  正常
           →   λ_t = 0          完全去耦
  - Conservative: λ_t = c=0.005 · γ_t · (1-δ_t)  正常
              →   λ_t = 0          完全去耦
验证：collapse 行为是否仍在 92% → 10% 量级？
```

**为什么关键**：
- Supervisor 明确指出这是"关键方法论异议"
- 如果你不修复它，审稿人会立即指出并要求重新做
- 修复后，"LR-WD 耦合是结构性的"论断变得无懈可击

**优先级标签**：**1B - MANDATORY**

---

### 改进 1C：表4 对齐数据溯源澄清【时间：0.5-1h】

**现状问题**：
- 表4 显示 δ_t ≈ 恒定（关键第3个洞察的基础）
- 不清楚：这些统计是来自完整 200-epoch 运行，还是从 20-epoch 试点数据的阶段边界外推？
- 如果是外推，对齐不变性的论断会被削弱

**行动**：
```
第一步（5分钟）：检查代码/日志，确认表4数据源
  - 如果来自完整运行：标记为 "Full-batch alignment, 200 epochs" ✓
  - 如果来自外推：立即重新计算
    （方法：完整200-epoch运行中，定期（如每10 epoch）采样完整梯度计算 δ_t）

第二步（如需重新计算，20-40 min）：
  - 在完整AADWD运行中插入"对齐检查点"
  - 计算完整批次对齐（而非mini-batch）以获得最精确的δ_t
  - 更新表4，标注数据来源
```

**为什么关键**：
- 这直接影响第3个核心洞察的可信度
- 审稿人会询问："你的δ_t数据是什么来源？"
- 预先澄清比被问时才解释要好得多

**优先级标签**：**1C - MANDATORY**

## 两个高优先级强化（优先级 2A-2B）

### 强化 2A：跨架构预算等价性验证【时间：1.5-3.5h（取决于选项）】

**现状**：
- 预算等价性仅在 CIFAR-10/ResNet-20 验证
- 理论框架宣称对"非凸 SGD"通用，但范围太窄

**行动**（选择一个，优先级递增）：
```
选项 A（最便宜，推荐）：CIFAR-100/ResNet-20
  时间：~1.5h
  步骤：
    1. 运行 AADWD（使用已调好的参数），提取 λ̄ = (1/T) ∑ λ_t
    2. 运行固定 WD at λ = λ̄（单seed即可）
    3. 验证准确率相等性（目标差异 < 0.15%）
    4. 报告结果：例 "CIFAR-100/ResNet-20: λ̄=0.0032, 
       AADWD 92.18% ≈ Fixed(λ=0.0032) 92.16%, Δ = 0.02%"

选项 B（中等成本）：CIFAR-10/VGG-16-BN
  时间：~1.5h
  理由：不同的网络架构增强说服力

选项 C（高成本，最有说服力）：CIFAR-100/ResNet-50
  时间：~3.5h
  理由：更大的模型，更接近实践规模
```

**为什么高优先级**：
- 直接增强"预算等价性是普遍现象"的说服力
- 相对低成本（只需要 1-2 次完整运行）
- Supervisor 建议："至少一个 ImageNet 验证"；这作为 CIFAR 阶段的中间步骤是合理的

**优先级标签**：**2A - HIGH IMPACT**

---

### 强化 2B：细粒度对齐分析 + 层级自适应 WD 对照【时间：1-1.5h】

**现状问题**：
- 对齐信号仅在全局级别分析
- 潜在反驳："你的定义太粗糙，层级或块级对齐可能更有变化"

**行动**：
```
第一部分（30 min）：分层对齐分析
  - ResNet-20 有 4 个 residual block
  - 计算 δ_t,layer for layer in [1, 2, 3, 4] 整个 200-epoch
  - 绘制：δ_t,l(t) 时间序列（6 条曲线：4 层 + 1 全局 + 背景）
  - 计算：Var_t(δ_t,l) / E_t(δ_t,l) for each layer
  - 结论：即使在层级上，相对变化也 < 15%（与全局相似）

第二部分（45 min）：层级自适应 WD 对照
  - 新方法：λ_t,l = c · γ_t · (1 - δ_t,l) for each layer l
  - 对照：
    1. 固定 WD（所有层相同）
    2. AADWD（全局对齐）
    3. Layered AADWD（层级对齐）
  - 结果表：表X
    | 方法 | 准确率 | vs. Fixed WD |
    |------|--------|------------|
    | Fixed WD | 92.54% | baseline |
    | AADWD | 92.37% | -0.17% |
    | Layered AADWD | 92.35% | -0.19% |
  - 结论：即使用细粒度对齐，动态 WD 仍无优势
```

**为什么高优先级**：
- 主动应对"对齐定义太粗糙"的反驳
- 表明你已经探索了这个替代方向，增加论文的周全感
- 时间成本相对低

**优先级标签**：**2B - HIGH IMPACT (defensive)**
