# 实证主义者视角：必需的实验改进

## 核心观点

论文的核心数据是健全的，但**实验设计的粗糙**和**样本规模的不足**削弱了可信度。这个视角系统地列举每个关键主张需要什么样的实验支持。

## 改进1：多 seed 实验（强制）

**当前状态**：仅 seed=42，边际差异 0.17%-0.49%，无法排除噪声

**需求**：
```
表1（主要对比）：7 个方法 × 5 seeds （42, 123, 456, 789, 1010）× 200 epochs
结果：每个单元格显示 mean ± 2σ（95% CI）
```

**需要验证的假设**：
- AADWD Conservative vs. Fixed WD：现在声称 -0.17%（AADWD 更差）
  - 需证明：这个差异确实来自 WD 动态失败，而非随机波动
  - 预期：95% CI 不重叠，或重叠 <20%
  
- Random-WD vs. AADWD Aggressive：现在声称 0.01% 差异（随机略优）
  - 这个差异太小，可能是噪声，需要证明统计显著性不存在

**时间成本**：~2.5 小时（包括数据整理）

**优先级**：**强制** — NeurIPS/ICML 标准

## 改进2：解耦实验的 c 值一致性（强制）

**当前问题**：
- 主实验：aggressive c=2.5, conservative c=0.005
- 解耦实验：aggressive c=0.25, conservative c=0.0005（各小 10 倍）

**需求**：
```
重新运行 decoupled variant 实验，使用与主实验相同的 c 值：
- 激进型：c=2.5, 然后完全去耦（λ_t=0）
- 保守型：c=0.005, 然后完全去耦
验证：collapse 行为是否仍然出现（应该出现）
```

**需要验证的假设**：
- H0（虚假）：collapse 来自 c 值相对太大，导致过度衰减
- H1（真实）：collapse 来自 LR-WD 耦合的结构性破裂，与 c 值的绝对大小无关

**预期结果**：
- 如果 H1 正确，即使用与主实验相同的 c，collapse 应该仍在 92% → 10% 的量级
- 在现有数据中可能已经能看到这一点；只需确认并报告

**时间成本**：~40 分钟（或更少，如果现有数据已足够）

**优先级**：**强制**

## 改进3：表4 对齐数据的完整性验证（强制）

**当前疑虑**：对齐统计是否来自完整 200-epoch 运行？

**需求**：
```
确认表4中的每个δ值的数据来源：
- 如果来自完整200-epoch运行：标记为"Full-batch alignment, 200 epochs"
- 如果来自20-epoch试点数据的阶段边界外推：立即重新计算
  （方法：完整运行中在第t步采样完整梯度，计算δ_t）
```

**需要验证的假设**：
- H0（虚假）：对齐统计来自部分数据或外推，δ_t 的恒定性可能是假迹象
- H1（真实）：对齐统计来自完整运行，δ_t 的恒定性是真实现象

**预期结果**：
- 如果重新计算，δ_t 的分布应与现有数据相似（但更精确）
- 如果它们不同，需要解释为什么

**时间成本**：
- 验证数据源：~20 分钟
- 如需重新计算：~50 分钟（取决于是否已有检查点）

**优先级**：**强制**

## 改进4：跨架构和数据集的预算等价性（高优先级）

**当前状态**：预算等价性仅在 CIFAR-10/ResNet-20 验证

**需求**：
```
在至少一个额外的组合上验证相同的现象：
选项 A：CIFAR-100/ResNet-20
选项 B：CIFAR-10/VGG-16-BN
选项 C：CIFAR-100/ResNet-50（较贵但更有说服力）

对于每个选项：
1. 运行 AADWD（任意参数），提取 λ̄ = (1/T) ∑ λ_t
2. 运行固定 WD at λ = λ̄（单 seed）
3. 报告准确率相等性（目标：差异 < 0.15%）
```

**需要验证的假设**：
- H0（虚假）：预算等价性是 CIFAR-10/ResNet-20 特有的巧合
- H1（真实）：这是跨架构和数据集的普遍现象

**时间成本**：
- 选项 A（最便宜）：~1.5 小时
- 选项 B：~1.5 小时
- 选项 C：~3.5 小时

**优先级**：**高** — 大幅增强核心发现的说服力

## 改进5：细粒度对齐分析（中优先级）

**当前状态**：仅全局 δ_t = |⟨∇f, w⟩| / (‖∇f‖ ‖w‖)

**需求**：
```
分层分析（Resnet-20 有4个残差块）：
- δ_t,layer for layer in [1,2,3,4]
- 绘制 δ_t,l(t) for each layer，对比全局 δ_t(t)
- 计算 Var_t(δ_t,l) / E_t(δ_t,l) for each layer
- 验证：即使在层级上，变化也很小
```

**新实验**：
```
尝试层级自适应 WD：
  λ_t,layer = c · γ_t · (1 - δ_t,layer)
与以下对照：
  1. 固定 WD（每层相同）
  2. AADWD（全局对齐）
  
预期：层级自适应仍不会超越固定 WD
```

**需要验证的假设**：
- H0（虚假）：对齐信号太粗糙；细粒度版本会更有效
- H1（真实）：即使在细粒度级别，对齐变化也太小，无法用于调制

**时间成本**：~1-1.5 小时

**优先级**：**中** — 加强论文对"为什么对齐信号无效"的论证

## 改进6：对照实验：其他动态 WD 方案（低-中优先级）

**当前状态**：尝试了 AADWD（对齐基）和 Random-WD

**新对照**：
```
A. 基于权重范数的 WD：
   λ_t = c · (1 - exp(-‖w_t‖²))
   直观：当权重开始爆炸时减少衰减
   
B. 基于梯度范数的 WD：
   λ_t = c · γ_t / (‖∇f_t‖ + ε)
   直观：在梯度大时减少衰减
   
C. AlphaDecay 的简化版：
   λ_t 每 epoch 重新计算一次（而非每步）
```

**需要验证**：即使用**不同的信号**，动态 WD 仍然无法超越固定 WD

**时间成本**：~1.5-2 小时

**优先级**：**低-中** — 有益但非关键。如果截止很近，可跳过。

## 改进7：超参数敏感性分析（中优先级）

**当前状态**：报告了最优 c（对于 AADWD），但缺少完整的扫描

**需求**：
```
CIFAR-10/ResNet-20 上的完整扫描：
- 固定 WD：λ in {1e-4, 5e-4, 1e-3, 5e-3, 1e-2, 5e-2}（6个值）
- AADWD 保守型：c in {0.001, 0.005, 0.01, 0.05, 0.1, 0.5, 1.0, 2.5}（8个值）

结果：绘制准确率 vs. 参数热力图（或线图）
分析：
  - 固定 WD 的最优点有多"尖"？
  - AADWD 的最优点有多"尖"？
  - 声称 AADWD "不敏感" 成立吗？
```

**时间成本**：~1.5-2 小时

**优先级**：**中** — 支持"AADWD 不提供鲁棒性优势"的论断

## 改进8：ImageNet 试点（可选，高价值）

**当前状态**：所有结果 ≤ CIFAR-100，理论声称通用性

**需求**：
```
单次 ImageNet-21k（或 ImageNet-1k subset）运行：
- ResNet-50（标准）
- 100 epochs SGD
- 测试：
  1. 预算等价性（extract λ̄ from AADWD, verify with fixed WD）
  2. LR-WD 耦合（改变学习率，验证 λ ∝ γ² 仍必要）
  3. 对齐恒定性（验证 Var_t(δ_t) 仍小）

预期：相同的三个负面结果应该持续
```

**时间成本**：~6-8 小时（取决于 GPU）

**优先级**：**可选，但高价值** — 如果有 GPU 时间，这会大幅增加论文的说服力

## 执行优先级总结

| 改进 | 类型 | 成本 | 优先级 | 备注 |
|------|------|------|--------|------|
| 多 seed (5x) | 必需 | 2.5h | 强制 | NeurIPS 标准 |
| 解耦 c 值一致性 | 必需 | 0.5h | 强制 | 关键方法论 |
| 表4 溯源验证 | 必需 | 0.5h | 强制 | 第3个洞察的基础 |
| 跨架构预算等价 | 强化 | 1.5h | 高 | 大幅增强说服力 |
| 细粒度对齐分析 | 强化 | 1.5h | 中 | 应对细粒度反驳 |
| 其他动态方案 | 强化 | 2h | 低-中 | 可选，增加严谨性 |
| 超参数敏感性 | 强化 | 2h | 中 | 支持鲁棒性论断 |
| ImageNet 试点 | 扩展 | 8h | 可选/高 | 如有GPU时间 |

## 完全实验计划（冲刺 2 天）

**Day 1（8-10 小时）**：
1. 解耦 c 值修复 (0.5h)
2. 表4 溯源验证 (0.5h)
3. 多 seed 实验启动 (2h，后台运行)
4. 跨架构预算等价性启动 (2h，后台运行)
5. 细粒度对齐分析代码 (1h)

**Day 2（4-6 小时）**：
1. 多 seed 和跨架构实验完成
2. 细粒度对齐分析 (1.5h)
3. 数据汇总、图表生成、论文更新 (1-2h)

## 结论

这些改进不是可选的"锦上添花"。前三个（多seed、c值一致、表4溯源）是**强制的** — 无论如何都要完成。后续改进按优先级递减。完成前三个后，论文已具有标准审稿人会接受的严谨性。完成至少到"跨架构验证"后，论文成为该领域的权威声明。
