## Critical Planning Gap: Data Pipeline Accuracy

A new planning gap has emerged: **the analysis pipeline produces incorrect numbers**. VGG Table 3 and NoBN Table 5 both contain errors when compared to raw epoch_metrics.jsonl data. This means either:

1. The analysis script has a bug (e.g., using a different "best accuracy" metric than epoch_metrics.jsonl reports)
2. The analysis script ran on an earlier/different version of the data
3. The numbers were manually entered or generated by a different process

This is a planning/infrastructure issue: the pipeline should guarantee that paper-reported numbers exactly match raw data. A "verify_figures.py" script was planned in the methodology (Section 8, Code Infrastructure) but has not been implemented.

**Recommendation**: Before any further writing, implement a data-paper consistency checker that:
1. Reads all epoch_metrics.jsonl files
2. Computes means/stds/spreads
3. Compares to the paper's markdown tables
4. Reports any discrepancy > 0.01%
