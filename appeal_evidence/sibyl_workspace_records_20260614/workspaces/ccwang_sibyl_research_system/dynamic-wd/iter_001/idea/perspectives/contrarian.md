# 反对者视角：质疑和反驳

## 核心观点

这篇论文的三个核心论断（预算等价性、LR-WD 耦合、δ_t 不变）虽然有数据支持，但都存在可被攻击的边界。本视角系统地列举最可能的反驳，并提出防御策略。

## 反驳1：预算等价性只是巧合

**攻击**："你从 AADWD 中提取 λ̄，然后用固定 WD 运行——当然会匹配，因为你挑选了结果！这不是预算等价性，这是**循环论证**。"

**防御**：
- 明确说明：λ̄ 是从独立的 AADWD 运行中 **a posteriori** 提取的，而非网格搜索 λ 找到的最佳值
- 展示：其他随机 λ 值（不等于 λ̄）会导致不同的准确率，验证 λ̄ 的特殊性
- 对照实验：从**随机的**动态 WD 方案（e.g., λ_t = c · sin(t)）提取时间平均，验证这些**不会**再现相同的准确率

**工作量**：一个额外的实验，~30 分钟

**优先级**：**高** — 这是最容易被攻击的claim。补强后论文防御能力大增。

## 反驳2：LR-WD 耦合只适用于小模型

**攻击**："你的所有实验都在 ResNet-20 和 VGG-16 上。在现代大模型上，LR 和 WD 可能独立调整——你的耦合论可能是小网络的假迹象。"

**防御**：
- 至少在一个大模型（e.g., ResNet-50 on ImageNet subset 或 ViT-Small）上验证 λ ∝ γ² 的必要性
- 不需要全 ImageNet，但需要相对更大的规模（≥1M 参数，≥几小时训练）
- 展示：即使在大模型上，打破 λ ∝ γ² 的关系也会导致性能下降

**工作量**：~6 小时（可能需要特殊硬件）

**优先级**：**中-高** — 这直接影响"普适性"的声称。

## 反驳3：δ_t 恒定只是因为你的对齐定义太粗糙

**攻击**："你用的是全局对齐 δ_t = |⟨∇f, w⟩| / (‖∇f‖ ‖w‖)。如果你看**层级对齐**或**参数块对齐**，也许会看到有意义的变化。你的动态 WD 失败只是因为定义选择不当。"

**防御**：
- 不仅报告全局 δ_t，还报告**层级 δ_t,l** 的时间序列（最多6层）
- 展示：即使在层级上，δ_t,l 的方差也很小（相对于平均值）
- 新实验：尝试**层级自适应 WD**，其中 λ_t,l = c · γ_t · (1 - δ_t,l)，验证这**仍然无法超越固定 WD**

**工作量**：一个完整的试点（~1.5h），代码修改（~30 min）

**优先级**：**中** — 表明你已经探索了"没有工作的替代定义"的想法，这加强了论文的可信度。

## 反驳4：你没有尝试足够多的动态策略

**攻击**："你尝试了 AADWD 和 random-WD，但世界上可能有其他动态 WD 方法你没想到。你的负面结论太绝对了。"

**防御**：
- 在论文中明确限定范围："我们验证了**基于梯度-参数对齐的动态 WD**的无效性。其他动态规则（e.g., 基于输出激活、损失曲率）超出本研究范围。"
- 添加一个新的对照实验：**谱基 WD**（如文献中的 AlphaDecay）与你的方法并列，验证它也无法超越固定 WD
- 在 Discussion 中讨论：为什么基于**其他信号**的动态规则可能在原则上更有希望

**工作量**：~1-2 小时（AlphaDecay 的快速实现和基准）

**优先级**：**低-中** — 有益但非关键。

## 反驳5：实验设计的其他混淆变量

**攻击**：
- "你在解耦实验中混用了不同的 c 值，凭什么我们应该相信 collapse 结果？"
- "表4 的对齐统计是否来自完整的 200-epoch 运行？如果是外推的，那 δ_t 不变的论证就虚弱了。"
- "你没有运行足够的 seed，小差异可能来自随机波动。"

**防御**：
这三个都已由 supervisor review 明确指出。
- **解耦 c 值**：重新用相同的 c 值运行（见pragmatist视角）
- **表4 数据溯源**：确认来自完整运行，或重新计算
- **多 seed**：运行 5 个 seed，报告置信区间（见pragmatist视角）

**工作量**：这些已列在pragmatist的优先级 2-3 中

**优先级**：**最高** — 这些不是高级智力挑战，而是标准的审稿人检查表。不解决它们会自动导致拒稿。

## 防守总结

| 反驳 | 防御成本 | 优先级 | 现状 |
|------|--------|--------|------|
| 预算等价性循环论 | 中 | 高 | 需补强 |
| 耦合仅限小模型 | 高 | 中-高 | 希望解决 |
| 对齐定义太粗糙 | 中 | 中 | 需补强 |
| 未尝试所有策略 | 中-高 | 低-中 | 可选 |
| 实验设计混淆 | 中 | 最高 | 强制 |

## 结论

前四个反驳都可以通过额外实验有效防御。最后一个（实验设计）是**强制的** — pragmatist视角已列出这些修复。完成pragmatist的优先级1-4后，论文的防御强度会显著提升，足以应对标准审稿人的审查。
